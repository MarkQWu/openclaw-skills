爆款剧本工坊

使用说明 · 零技术门槛 · 只需 2 个命令

只记 2 个命令就够

我想做输入
开始 / 继续写剧本/开始
另开一本新剧本/新建 剧名

其他都可以忘。

场景 A · 第一次用

你 → /开始 Claude → 看起来你还没有剧本。项目名？ 你 → 都市神医 Claude → 已创建项目《都市神医》。 这个故事给谁看？男性向 / 女性向 / 男女通吃？

（接下来按 Claude 一步步引导走完选题即可）

场景 B · 同时写两本剧本

已经在写《我的剧本》，想开第二本：

你 → /新建 都市神医 Claude → 项目《都市神医》已创建。输入 /开始 进入 你 → /开始 Claude → 你的剧本： 1. 《我的剧本》分集 25/30（2天前） 2. 《都市神医》未开始（刚刚） 3. 新建一本剧本 回复数字或新剧名 你 → 2 Claude → 欢迎来到《都市神医》，我们来定方向。

两本互不干扰，随时 /开始 再选数字即可切换。

场景 C · 回来继续写

你 → /开始 Claude → 你的剧本： 1. 《我的剧本》分集 25/30（2天前） 2. 《都市神医》创作方案（刚刚） 3. 新建一本剧本 回复数字或新剧名 你 → 1 Claude → 欢迎回来：《我的剧本》，阶段 分集，进度 25/30。 继续写 /分集 26 你 → /分集 26

常用命令速查

/开始 总入口：列出所有剧本让你选（继续/新建）
/新建 剧名 直接新建一本新剧本
/分集 1 写第 1 集（批量 /分集 5-8，续写 /分集 next）
/自检 1 质检第 1 集（自动修复 /自检 1 --fix）
/合规 国内发行合规审核
/导出 打包完整剧本，直接生成 Word（.docx）；强制重算梗概: --force-resynth
/导出 N 只导出第 N 集，直接生成单集 Word（.docx）
/项目列表 看所有剧本进度一览
/选题会 召集跨域专家对题材做结构性碰撞，产出创作处方
/圆桌诊断 1 对已自检的第 1 集做三视角质量诊断（需先跑 /自检 1）
/查看碰撞 查看已保存的碰撞记录和处方
/查看诊断 [N] 查看已保存的圆桌诊断记录
/使用说明 打开本使用说明
/帮助 完整命令列表

常见问题

Q1. 我刚 /新建 了，但 /开始 列表找不到？
确认项目名拼写。再输 /项目列表 看所有剧本。
Q2. 我在 WorkBuddy Windows 用，需要切换工作目录吗？
不需要。v1.13.0 起所有剧本自动存到统一位置 ~/short-drama-projects/（Windows：C:\Users\你的用户名\short-drama-projects\）。你只管用 /开始 和 /新建，Claude 帮你处理路径。
Q3. 我以前用 v1.10 或 v1.12 写过剧本，升级后剧本会丢吗？
不会。首次 /开始 时，Claude 会检测到你的老剧本，询问：
  • [1] 就地继续（兼容模式，以后不会在 /项目列表 里出现）
  • [2] 迁移到新位置（推荐，以后多剧本切换更方便）
选 [2] Claude 会自动把剧本文件移到新位置，集数、人物、配置全部保留。
Q4. /新建 我的剧本，但已经有同名项目怎么办？
Claude 会拒绝并提示：《我的剧本》已存在（阶段 X, 进度 N/M）。继续用 /开始 选该项目，或换新名。防止误覆盖。
Q5. 项目名有什么规则？

✅ 可以：中文、数字、英文、空格、基本标点

❌ 不能：斜杠 / \、冒号 :、问号 ?、星号 *、引号 "、尖括号 < >、竖线 |、单独的 . 或 ..

长度超 50 字符会警告但可确认。

Q6. 我不小心删了 .drama-state.json 怎么办？
集数内容（episodes/*.md）还在 → 可以手动重写 state（联系作者协助）。只是迷失了进度感 → /开始 + 选该项目，Claude 会根据 episodes/ 目录重建进度感。
Q7. /更新 拉不到新版本？
检查网络 + 确认 skill 目录可写。还不行联系作者。
Q8. 我想把剧本 A 的某一集复制到剧本 B？
目前不支持跨剧本复制。手动去 ~/short-drama-projects/剧本A/episodes/ 下找对应 ep00X.md 复制。

剧本文件都存在哪？

统一位置：~/short-drama-projects/<剧名>/

每本剧包含：

剧名/
├── .drama-state.json     # 进度（自动维护，别删）
├── brainstorm.md         # 构思记录
├── creative-plan.md      # 创作方案
├── characters.md         # 角色档案
├── episode-directory.md  # 分集目录
├── episodes/             # 分集剧本
│   ├── ep001.md
│   └── ...
├── export/               # 导出文件
│   └── 剧名-完整剧本.docx
└── README.md             # 项目自述（/项目状态 生成）

想备份某本剧？直接复制整个剧名文件夹即可。

核心心法

  • 不用记路径：所有剧本自动在 ~/short-drama-projects/
  • 不用切目录：/开始 让 Claude 帮你定位
  • 不怕多本剧：/开始 列表选数字秒切换
  • 不怕同名误覆盖：/新建 自动拒绝重名在建项目

有问题看 /帮助 或联系作者。
