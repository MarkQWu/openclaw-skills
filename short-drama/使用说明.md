# 爆款剧本工坊 · 使用说明

## 只记 2 个命令就够

| 我想做 | 输入 |
|---|---|
| 开始 / 继续写剧本 | `/开始` |
| 另开一本新剧本 | `/新建 剧名` |

其他都可以忘。

---

## 场景 A · 第一次用

```
你 → /开始

Claude → 看起来你还没有剧本。项目名？

你 → 都市神医

Claude → 已创建项目《都市神医》。
        这个故事给谁看？男性向 / 女性向 / 男女通吃？

（接下来按 Claude 一步步引导走完选题即可）
```

## 场景 B · 同时写两本剧本

已经在写《我的剧本》，想开第二本：

```
你 → /新建 都市神医

Claude → 项目《都市神医》已创建。输入 /开始 进入

你 → /开始

Claude → 你的剧本：
        1. 《我的剧本》分集 25/30（2天前）
        2. 《都市神医》未开始（刚刚）
        3. 新建一本剧本

        回复数字或新剧名

你 → 2

Claude → 欢迎来到《都市神医》，我们来定方向。
         这个故事给谁看？...
```

两本互不干扰，随时 `/开始` 再选数字即可切换。

## 场景 D · 看到爆款想做同类型

看到爆款剧想快速做几个同赛道变体——`/重构` 提骨架换皮：

```
你 → /重构 《我的前半生》

Claude → ## 骨架提取：《我的前半生》
         开局钩：全职太太遭出轨，身份落差启动
         捶杀机：丈夫+小三+婆家三方联手排挤
         倒戈点：闺蜜/贵人出手相助，女主职场觉醒
         绝杀式：事业翻身 + 情感选择主动权收回

         ## 同构变体
         ### 变体 1：古装宫廷
         被废皇后借娘家旧部东山再起，重掌后宫

         ### 变体 2：战神归来
         被家族逐出的继承人回归，借军功反打三方

         ### 变体 3：修仙
         师门叛徒假死归来，拉旧部清理门户

你 → 选变体 2

你 → /新建 战神归来
     （接下来走 /开始 正常流程走选题、人物、分集）
```

**用途**：同构创作——爆款换皮、现象级小说改编短剧、题材迁移测试。

**注意**：`/重构` 只产出创意骨架**不建项目文件**。选中变体后用 `/新建` 建项目、`/策划` 展开。

## 场景 E · 仿写爆款（大文件自动处理）

手里有一部 60 集的爆款剧（.docx），想做同赛道变体——`/仿写` 直接提路径，50 集以上自动压缩：

```
你 → /仿写 ~/Downloads/5_《女相师》1-60集.docx

Claude → 源剧较大（60 集），正在自动压缩...
         ✅ 已压缩（前 10 集保留全文 + 其余结构摘要），节省约 66% token。

         ## 骨架提取：《女相师》
         身份密码：平民命理师，身份借壳权贵后宅
         压迫机制：相府三方（嫡母 / 庶兄 / 侯爷党羽）联手打压
         ...（6 维 DNA 提取完毕）

         这个仿写项目叫什么名字？

你 → 穿越女国师

Claude → 项目《穿越女国师》已建立。
         当代热点推荐：玄学国潮、AI 算命热、古今对照反差...
         （完成选题融合后：/仿写 1 开始写第 1 集）
```

**适合**：手里有完整剧本文件想快速做同类型变体。文件超过 50 集不用拆分，自动处理。

## 常用命令速查

| 命令 | 做什么 |
|---|---|
| `/开始` | 总入口：列出所有剧本让你选（继续/新建） |
| `/新建 剧名` | 直接新建一本新剧本 |
| `/重构 <参考剧本>` | 从爆款提取四步骨架，生成 N 个同构变体（仅创意骨架，不建项目） |
| `/仿写 [参考剧本]` | 仿写爆款：提取结构 DNA → 融入热点 → 逐集生成正文；50 集以上自动压缩 |
| `/分集 1` | 写第 1 集（批量：`/分集 5-8`；续写：`/分集 next`） |
| `/自检 1` | 质检第 1 集（自动修复：`/自检 1 --fix`） |
| `/合规` | 国内发行合规审核 |
| `/导出` | 打包完整剧本，直接生成 Word（.docx）；强制重算梗概：`--force-resynth` |
| `/导出 N` | 只导出第 N 集，直接生成单集 Word（.docx） |
| `/项目列表` | 看所有剧本进度一览 |
| `/选题会` | 召集跨域专家对题材做结构性碰撞，产出创作处方 |
| `/圆桌诊断 1` | 对已自检的第 1 集做三视角质量诊断（需先跑 `/自检 1`） |
| `/查看碰撞` | 查看已保存的碰撞记录和处方 |
| `/查看诊断 [N]` | 查看已保存的圆桌诊断记录 |
| `/角色一致性` | 跨集角色一致性审查（性格/外貌/动机/称呼漂移；支持指定集数范围或角色名） |
| `/帮助` | 完整命令列表 |

## 常见问题

### Q1. 我刚 `/新建` 了，但 `/开始` 列表找不到？

确认项目名拼写。再输 `/项目列表` 看所有剧本。

### Q2. 我想把剧本 A 的某一集复制到剧本 B？

目前不支持跨剧本复制。手动去 `~/short-drama-projects/剧本A/episodes/` 下找对应 `ep00X.md` 复制。

### Q3. 我在 WorkBuddy Windows 用，需要切换工作目录吗？

**不需要**。v1.13.0 起所有剧本自动存到统一位置 `~/short-drama-projects/`（Windows：`C:\Users\你的用户名\short-drama-projects\`）。你只管用 `/开始` 和 `/新建`，Claude 帮你处理路径。

### Q4. 我以前用 v1.10 或 v1.12 写过剧本，升级后剧本会丢吗？

不会。首次 `/开始` 时，Claude 会检测到你的老剧本，询问：

- **[1] 就地继续**（兼容模式，以后不会在 `/项目列表` 里出现）
- **[2] 迁移到新位置**（推荐，以后多剧本切换更方便）

选 [2] Claude 会自动把剧本文件移到新位置，你原有的集数、人物、配置全部保留。

### Q5. `/新建 我的剧本`，但已经有同名项目怎么办？

Claude 会拒绝并提示：`《我的剧本》已存在（阶段 X, 进度 N/M）。继续用 /开始 选该项目，或换新名`。防止误覆盖。

### Q6. `/仿写` 参考剧本有 60 集，文件很大，需要手动拆分吗？

不需要。v1.30.0 起 `/仿写` 会自动检测文件大小：超过 50 集时调用内置压缩脚本，前 10 集保留原文、其余集生成结构摘要，整个过程自动完成。直接提供 `.docx`、`.md` 或 `.txt` 路径即可。

### Q7. 项目名有什么规则？

- ✅ 中文、数字、英文、空格、基本标点
- ❌ 斜杠 `/` `\`、冒号 `:`、问号 `?`、星号 `*`、引号 `"`、尖括号 `<` `>`、竖线 `|`
- ❌ 单独的 `.` 或 `..`
- 长度超 50 字符会警告但可确认

### Q8. 我不小心删了 `.drama-state.json`？

- 集数内容（`episodes/*.md`）还在 → 可以手动重写 state（联系作者协助）
- 只是迷失了进度感 → `/开始` + 选该项目，Claude 会根据 `episodes/` 目录重建进度感

### Q9. `/更新` 拉不到新版本？

检查网络 + 确认 `~/.claude/skills/short-drama/` 或 `~/.workbuddy/skills/short-drama/` 目录可写。还不行联系作者。

### Q10. `/开始` 老项目时被问"请选择承制介质"，回 1 还是 2？

新版本（v1.16+）引入了承制介质字段，区分**仿真人 AI 剧** vs **漫剧** —— 两者的剧本格式硬规则不同（场景数 / 单条台词长度 / 是否允许微表情等）。老项目首次加载时缺这个字段，会被一次性询问：

```
1）仿真人 AI 剧
2）漫剧（AI 漫剧或动态漫画）
```

按你这本剧实际**承制形式**选：

- 用真人演员（或 AI 仿真人视频）拍摄 → **回 1**
- 用动态漫画 / AI 漫剧形式承制 → **回 2**

回错了不致命——可以手动改 `~/short-drama-projects/<剧名>/.drama-state.json` 里的 `medium` 字段值（`"ai_live"` 或 `"comic"`）。后续命令（`/分集` `/自检` 等）会按 medium 自动加载对应硬规则。

新建项目走 `/开始` 流程时会在 Step 6「推荐配置确认」里一并选，不会再触发这个询问。

## 剧本文件都存在哪？

统一位置：`~/short-drama-projects/<剧名>/`

每本剧包含：

```
剧名/
├── .drama-state.json     # 进度（自动维护，别删）
├── brainstorm.md         # 构思记录
├── creative-plan.md      # 创作方案
├── characters.md         # 角色档案
├── episode-directory.md  # 分集目录
├── episodes/             # 分集剧本
│   ├── ep001.md
│   └── ...
├── export/               # 导出文件
│   └── 剧名-完整剧本.docx
└── README.md             # 项目自述（/项目状态 生成）
```

想备份某本剧？直接复制整个 `剧名/` 文件夹即可。

## 核心心法

- **不用记路径**：所有剧本自动在 `~/short-drama-projects/`
- **不用切目录**：`/开始` 让 Claude 帮你定位
- **不怕多本剧**：`/开始` 列表选数字秒切换
- **不怕同名误覆盖**：`/新建` 自动拒绝重名在建项目

有问题看 `/帮助` 或联系作者。
