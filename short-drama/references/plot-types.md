# 情节类型库（40 种）

## 核心理念：情节结构论 vs 爽点结果论

写剧本有两个独立问题：**故事发生了什么** vs **观众为什么爽**。本文件服务于前者。

- **情节类型**（本文件）= 故事结构坐标（背叛 / 拯救 / 身份错位…）= 问"这个故事讲什么"
- **爽点类型**（`satisfaction-matrix.md`）= 观众情绪结果（身份碾压 / 打脸复仇 / 情感爆发…）= 问"观众为什么停不下来"

两者正交。一个"背叛 + 陷入麻烦 + 拯救"的故事可以通过身份碾压爽点解决，也可以通过情感爆发爽点解决。选好情节结构再选爽点设计。

**使用时机**：`/创作方案` 阶段的"一句话故事线 + 核心冲突"生成时，从 40 种里挑 2-5 个组合成 1+n 故事类型（见末尾"组合公式"段）。

## 跨阶段伏笔闭合（复仇类型进阶变体）
<!-- 来源：Gemini方法论融合 2026-04-23 -->

普通复仇 = 受害 → 积累 → 反击（线性）
**伏笔式复仇 = 第1集埋雷 → 中段读者遗忘 → 终局引爆（非线性）**

| 伏笔类型 | 埋入时机 | 闭合时机 | 效果 |
|---------|---------|---------|------|
| 断绝型（主动布局）| 第1-3集主角主动布局 | 第19-24集引爆 | 观众恍然大悟"原来早就安排了" |
| 被动型（reactive）| 第7-12集主角被迫留下 | 第13-18集被反派触发 | 反派激活了自己的失败 |
| 物品型 | 第1集出现的道具/信件/记录 | 终集成为铁证 | 视觉闭环，极强满足感 |

**断绝型（主动布局）写法要点：**
- 第1-3集主角做一件"看起来认命/让步"的事（实则布局）
- 第7-15集反派以为自己已经赢了
- 第19集+：主角揭示这个"认命"是陷阱

**与普通复仇的区别：** 伏笔式复仇需要在第1集就写好闭合点，反推设计，不能中途加。

## 组合公式：故事类型 1+n（n ≥ 2 且 ≤ 5）

单一情节类型支撑不了 50-100 集。爆款剧本方法 = 从 40 种里组合 2-5 个。

### 经典组合范例

**范例 1：背叛 + 陷入麻烦 + 拯救 + 奇遇**
> 男主发现未婚妻偷情（背叛）被打（陷入麻烦），濒死时被女主拯救（拯救），因玉佩获得玄学道术（奇遇）。—— 《龙跃鹤唳》

**范例 2：背叛 + 身份错位 + 陷入麻烦**
> 男主惨遭兄弟背叛（背叛），灵魂穿越到傻子身上（身份错位），被逼迫和一个傻子传宗接代（陷入麻烦）。

**范例 3：联姻 + 替身 + 伤病 + 陷入麻烦**
> 女主被迫嫁给坐在轮椅上的男主（联姻 + 伤病），卷入豪门阴谋却发现男主并未残疾，而男主竟把自己当成替代品（替身），女主求放过却被男主牢牢禁锢（陷入麻烦）。—— 《总裁的替嫁娇妻》

### 组合原则

1. **n ≥ 2**：至少 2 个才能撑起主线 + 支线的复杂度
2. **n ≤ 5**：超过 5 个观众记不住、编剧也容易写乱
3. **必含 1 个"开头放置型"**：拯救 / 陷入麻烦 / 追击 —— 用来给开头下压情绪的钩子
4. **主线与副线要跨大类**：比如"背叛（情感关系）+ 奇遇（获得某物）+ 复仇（人际关系）"比"背叛 + 误会 + 恋爱"（全在情感关系类内）更有层次

### 反模式

- 只选 1 个类型（单薄，无法撑 50 集）
- 选 6+ 个（信息过载，观众跟不上）
- 全堆在同一大类（层次单一，容易枯燥）
- 没有"开头放置型"类型（开头无钩子，观众流失）

## 创新的本质：组合不是复制

情节可以俗套，但必须**俗套的与众不同**。如男频"男主的女友出轨"很老套，**可以变地点**（订婚宴 / 直播 / 跟男主的弟弟 / 父亲）、**变方式**（一对多 / 一对特定关系人）、**变情景**（事后被主角撞见 vs 主角提前安装摄像头自己看）、**变角色**（闺蜜 vs 亲妹 vs 青梅）。

**不会创新 = 积累不够**。遇到"写不出新花样"时不是天赋问题，是见过的组合太少。对标同类型剧本是唯一办法。
