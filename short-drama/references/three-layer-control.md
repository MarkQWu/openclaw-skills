# 原创短剧三层控制模型

本文件定义原创 `short-drama` 的工程化边界：哪些规则必须 100% 执行，哪些只锁故事功能，哪些只做创作建议。它不是创作模板，而是防止两种事故：

1. 把地基层问题当成创作自由，导致项目崩盘。
2. 把血肉层偏好当成硬规则，导致剧本模板化、无作者感。

核心原则：**按崩盘风险决定控制强度，不按主观重要程度决定控制强度。**

## 控制强度

这些比例不是相加为 100 的分配表，而是每层可被工程规则控制的强度。

| 层级 | 原创短剧默认控制强度 | 原因 |
|---|---:|---|
| 地基层 | 100% | 负责活下去：项目状态、合规、格式、媒介、连续性、事实底线。一处松动会污染全书或阻断交付。 |
| 骨架层 | 75% | 负责看下去：原创没有可继承的参考骨架，受众、题材、首集、主线、付费卡点、爽点和尾钩必须强锁功能。 |
| 血肉层 | 35% | 负责有灵魂：原创正文全由模型生成，台词、动作、场面、节奏颗粒和记忆点需要更强质量防线，但仍不能模板填空。 |

为什么原创不是 `60% / 20%`：

- 原创没有参考剧本的既有商业骨架兜底，商业成败主要发生在骨架层，所以从 60% 提到 75%。
- 原创正文完全由模型生成，AI Slop、不可拍、对白失真、爽点落地失败会直接影响留存，所以血肉层从 20% 提到 35%。
- 这两个数值仍然不是硬模板。骨架层锁 story job、pressure、payoff、hook，不锁场景表皮；血肉层强化评分、诊断和重写建议，不因风格偏好单独阻断。

## 第一层：地基层 - 100% 工程化

地基层保护一本剧能否成立、能否交付、能否持续写下去。这里没有创作自由。

### 锁定内容

- 项目状态：活跃项目、剧名、集数、受众、题材、语言、`mode`、`medium`、已完成集数、质量分数、当前文件路径。
- 世界观和设定事实：`setting-bible.md`、专业事实、时代规则、地理/组织/职业边界、已经确认的世界规则。
- 人物核心：主角和核心配角的身份、动机、欲望-恐惧对位、关系逻辑、能力边界、称呼关系、voice 禁区。
- 平台和合规红线：违法、平台风险、敏感题材、文化禁区、出海 hard rules、国内/海外对白格式分化。
- 承制介质硬约束：`ai_live` / `comic` 的场次数、可拍性、场景头、台词长度、镜头/分镜格式。
- 全书连续性：已发生事件、已承诺钩子、已埋伏笔、已回收爽点、人物关系变化、跨集台词复用记录。
- 输出和写入安全：命令输出格式、文件命名、Read-Modify-Write、导出门控、版本更新机制。

### 阻断条件

命中以下情况时，必须阻断继续生成、导出、交付建议或发布建议，并给出证据、责任归属和最小修复动作：

- 命令将读写错误项目、错误集数或覆盖未知 state。
- 剧本违反合规、平台、市场或 `mode` 专属硬规则。
- 厚型/中型题材编造不可追溯事实，或绕过 `setting-bible.md`。
- 角色核心、关系状态、世界规则、已发生事件被无依据改写。
- 介质硬约束失效，导致不可拍、不可画、不可导出或格式不合格。
- `/导出` 遇到已自检且不合格的集数。
- 血肉层修改会反向破坏上述任一地基层事实。

## 第二层：骨架层 - 75% 工程化

骨架层保护短剧的商业引擎。它锁定“这一段故事必须完成什么功能”，但不规定“必须用哪句台词、哪个道具、哪个场景完成”。

原则：**锁功能，不锁表皮；锁节点，不锁写法。**

### 锁定内容

- 受众和题材定位：男性向 / 女性向 / 男女通吃，题材、基调、结局、集数和商业模式。
- 故事主线：一句话故事线、核心冲突、主角目标、主要反派压力、最终承诺。
- 首集功能：开场冲突、主角痛点、压迫者动作、信息差、完整爽点弧、尾钩和第 2 集承接。
- 分集功能：每集核心事件、核心爽点、观众情绪、推进职责、尾钩职责、承接义务。
- 付费卡点：付费集位置、付费前压力、付费后兑现、IAP/IAA/混合模式差异。
- 爽点系统：爽点类型、痛点铺垫、释放强度、余震、主线连续性。
- 节奏曲线：前 10 集密度、关键集比例、阶段高潮、全剧推进速度。
- 模式适配：国内/出海、`ai_live`/`comic` 的结构性差异。

### 可自由实现

- 场景地点、道具、冲突动作、证据物、误会形式、反派出招方式。
- 同一爽点功能下的具体打脸方式、浪漫推进方式、悬疑揭示方式、喜剧包袱方式。
- 尾钩的具体画面和台词，只要它完成“下一秒谁要做什么”的功能。
- 场内情绪递进的细节顺序，只要不破坏本集的锁定职责。

### 骨架契约

`/策划`、`/分集目录`、`/分集` 产出的结构性内容，应能回答下列字段。缺字段不一定阻断，但若缺失导致后续无法写正文，应先补骨架再写血肉。

| 字段 | 含义 |
|---|---|
| `locked_audience` | 本剧主要给谁看，不能在中途漂移。 |
| `locked_genre_promise` | 题材向观众承诺的核心爽感。 |
| `locked_main_conflict` | 全剧主冲突和压迫来源。 |
| `locked_episode_job` | 本集在观看链条里的职责。 |
| `locked_viewer_emotion` | 本集主要要打中的观众情绪。 |
| `locked_satisfaction` | 本集必须兑现或铺垫的爽点。 |
| `locked_exit_hook` | 本集末尾把观众拉到下一集的问题或动作。 |
| `free_execution_zone` | 可自由替换的场景、动作、道具、台词和表达方式。 |

## 第三层：血肉层 - 35% 工程化

血肉层保护剧本的生命感、作者感和可记忆性。它需要最低限度质量防线，但不能被流程表格吃掉。

### 保持自由

- 具体对白措辞、沉默、停顿、反应、动作细节和句式节奏。
- 同一剧情功能的幽默、羞辱、暧昧、痛感、悬疑、爽感实现方式。
- 一个角色如何回避真话、如何嘴硬、如何露怯、如何失控。
- 场面里的具体视觉记忆点、动作记忆点、声音记忆点。
- 文风、密度、长短句搭配和个人表达习惯。

### 轻约束检查

血肉层默认只做建议或 `needs_revision`，不单独阻断，除非它越界破坏地基层或骨架层。

- 是否明显 OOC 到人物关系或动机崩坏。
- 是否不可拍、不可画，或违反当前 `medium`。
- 是否出现合规、事实、格式、市场硬规则问题。
- 是否没有完成本集锁定的故事职责。
- 是否明显 AI 腔、套话、空泛情绪词、Trailer-Speak。
- 是否能说出一个观众记得住的具体瞬间。

### 禁止做法

- 不用固定台词模板替代角色判断。
- 不规定每集必须有固定数量的比喻、停顿、反转句、金句或镜头术语。
- 不因为“台词不够漂亮”“句式不够统一”“风格不够像某个参考”阻断正文生成。
- 不把圆桌诊断或自检里的审美建议升级为地基层硬规则，除非它修复的是 OOC、合规、事实、格式、媒介或骨架职责失败。

## 命令映射

| 命令/阶段 | 地基层 | 骨架层 | 血肉层 |
|---|---|---|---|
| `/开始` | 项目 state、受众、mode、medium、语言 | 题材和商业承诺初始定位 | seed 想象力自由 |
| `/策划` | 已确认配置、事实底线、模式边界 | 主线、三幕、付费卡点、爽点矩阵、结局承诺 | 题材表达、世界味道、情绪色调 |
| `/角色开发` | 人物核心、关系逻辑、能力边界、称呼关系 | 人物在主线和爽点系统中的功能 | voice 样本、缺陷质感、互动细节 |
| `/考据` | 事实可追溯、bible scope、虚构白名单 | 专业知识如何服务冲突和爽点 | 专业语感和细节选择 |
| `/分集目录` | 集数、关键/付费比例、连续性 | 每集职责、阶段节奏、关键节点 | 集标题和一句话表达 |
| `/分集` | state、bible、人物核心、medium、mode、格式、used-lines | 本集职责、爽点、尾钩、承接 | 台词、动作、场面、节奏颗粒、记忆点 |
| `/自检` | hard gates、导出阻断、证据质量 | 商业生命力、节奏、爽点、主线 | 台词真实感、AI Slop、可记忆瞬间 |
| `/圆桌诊断` | 不覆盖 state、不解锁未通过 gate | 诊断结构问题和修复优先级 | 提供创作建议，不自动升级为硬规则 |
| `/角色一致性` | 年龄、身份、动机、关系状态矛盾 | 弧线、称呼、阶段状态漂移 | 口吻鲜明度和互动质感 |
| `/导出` | 自检结果、格式、文件、缓存、docx 合法性 | 梗概综合保留主线和承诺 | 人物小传润色只改体裁，不增删事实 |
| `/分镜` | 角色卡、正文边界、prompt 格式、媒介可执行底线 | 镜头承接本集 story job 和情绪功能 | 景别节奏、画面颗粒、prompt 质感 |

## 冲突处理

1. 地基层与任何层冲突时，地基层胜出。
2. 骨架层与血肉层冲突时，先问血肉层是否还能完成锁定功能；能完成则保留血肉层自由，不能完成则修骨架执行。
3. 国内模式与出海模式冲突时，`mode=overseas` 读取 `references/overseas/`，不得机械套用国内首集、爽点、身份或合规口径。
4. `medium=ai_live` 与 `medium=comic` 冲突时，以当前 `.drama-state.json#medium` 的承制介质规则为准。
5. 自检评分不能洗白 hard gate；高分也不能覆盖阻断结论。
6. 低分 craft 项不能自动阻断导出，除非它触发地基层或骨架层失败。

## 输出要求

当命令需要说明阻断或修改建议时，按层级标注原因：

- `[地基层阻断]`：必须修，不能继续或不能交付。
- `[骨架层修复]`：先补结构职责，再写正文或继续下一阶段。
- `[血肉层建议]`：可修改以提高质量，但不当作硬阻断。

任何把血肉层问题升级为阻断的判断，都必须说明它破坏了哪一条地基层事实或骨架层职责。
