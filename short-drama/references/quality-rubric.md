# 质量自检评分标准（Rubric）

> 来源：行业编剧方法论提炼 + 实践。`/自检` 命令加载本文件。
> All Rights Reserved — 本文件为方法论知识层，未经许可不得复制分发。

## 评分方法论

1. **先推理再评分**：每个维度先列出从剧本中观察到的具体事实，再对照锚点给分
2. **整数分制**：不允许小数
3. **锚点校准**：参照各维度的分数锚点描述，找到最接近的锚点
4. **硬约束优先**：触发硬约束时分数受上限限制，无论其他表现多好

## 分值分配（总分动态：厚型/中型 80，轻型 70）

| 维度 | 分值 | 类型 | 说明 |
|------|------|------|------|
| 节奏 | 1-10 | 核心 | — |
| 爽点 | 1-15 | 核心（加权） | 短剧商业成功的核心驱动力，权重最高 |
| 台词 | 1-10 | 核心 | — |
| 格式与可拍性 | 1-5 | 门槛 | 达标即可，不是区分因素 |
| 主线与连贯性 | 1-10 | 核心 | — |
| 反抽象与镜头化 | 1-10 | 核心 | — |
| AI Slop | 1-10 | 核心 | — |
| 考据可追溯性 | 1-10 | 题材分化 | 厚型/中型计入总分；轻型记 N/A 不计入（题材分类见 genre-guide.md） |

**为什么爽点加权**：格式规范是必要条件但不区分好坏——格式差不过稿，格式好不代表剧本好。爽点密度和质量才是短剧留存率和商业表现的核心驱动力。

**为什么考据独立维度**：作为爽点子项会被稀释；作为独立维度才能落实「绝不编造事实」硬规则。轻型题材（甜宠/萌宝/喜剧等情绪型）不需要专业知识底座，故记 N/A。

## 一、节奏

### 检查项

#### 开场检查（第03讲）

- [ ] 人设建立：身份/地位/情绪通过具体事件展示，非旁白说明
- [ ] 主线目标：目标在前两集直白展现，不藏不绕
- [ ] 困境代入：即时、具体、有压迫感的危机
- [ ] 情绪推进：密集戏剧性情绪转换（非平铺直叙）
- [ ] 信息密度：台词承载多重功能（推剧情+揭关系+暴矛盾+强人设）

#### 开场禁忌（第03讲）

- [ ] 无旁白/内心独白铺背景
- [ ] 无冗余对话浪费开场时间
- [ ] 无平庸温吞开场（照搬原著式推进）
- [ ] 人设三问清晰：我是谁？要什么？面临什么危机？

#### ⚡ 三锚点检查（硬约束）

- [ ] 0-3s 冲突点：前 3 段内出现冲突/钩子
- [ ] 30s 情绪爆破：第 2-3 场次存在情绪高点
- [ ] 结尾钩子：有悬念标记（张力台词或冲击力画面）
- **缺少任意一个锚点 → 该维度 ≤5 分**

**题材适配**：甜宠/日常向的"0-3s 冲突点"可替换为"0-3s 情感钩子"（如心动瞬间、误会引发、反差萌）。核心是前 3 段内抓住观众，不限于冲突。

#### 单集节奏（第05讲）

- [ ] 前 30 秒：背景+困境+动机三合一（「前30秒定生死」）
- [ ] 中间 1 分钟：主线推动 + 人物关系 + 新伏笔
- [ ] 最后 30 秒：强悬念/未完成动作/反转
- [ ] 无"信息真空"过渡场景
- [ ] 无无效嘴炮拖沓段落
- [ ] 每集有实质性进展（非重复停滞）

#### 结尾钩子（第06讲）

- [ ] 钩子类型明确：张力台词（未说完的质问/身份暗示/情绪化宣言）或冲击力画面（对视未开口/手伸未触/门推一半）
- [ ] 遵循"高潮切开，一半留下集"原则
- [ ] 无强行加戏或内心独白复述目标
- [ ] 钩子是情节到高潮临界的自然停顿

#### 全剧大节奏（仅 /自检 all 时检查，第05讲）

- [ ] 1-10 集黄金窗口：主线确立 + 核心看点亮出 + 观看动机建立
- [ ] 11-30 集：主线实质推进，无重复停滞，适度引入支线
- [ ] 31-80 集：伏笔揭晓，多轮反转，情绪集中释放
- [ ] 题材节奏匹配：复仇/逆袭→快狠准；甜宠→误会-吃醋-和解-升温

### 分数锚点

| 分数 | 描述 |
|------|------|
| 2 | 开场超过 5 段才出冲突，中段有信息真空，结尾无钩子；三锚点缺失 2+ 个；用旁白铺背景 |
| 5 | 开场有冲突但五要素缺 2+，或前 30 秒未完成三合一；中段有拖沓嘴炮；结尾钩子力度弱（内心独白复述） |
| 8 | 五要素齐全，前 30 秒高效完成三合一；节奏紧凑无一处拖沓；三锚点全部达标；钩子有效 |
| 10 | 开场前 3 段即强冲突+多重反转，中段层层递进如多米诺骨牌，结尾钩子让人「划不走」；节奏如过山车 |

### 避坑（低分特征，第03/05讲）

- 节奏拖沓：短剧一集仅 500-800 字，不在"展开"而在"爆破"
- 受众错位：写情绪前先问「写给谁看」
- 情绪空洞：机械套用下跪/扇巴掌三件套（观众共情的是「尊严被践踏」不是具体行为）
- 桥段窠臼：不反复使用同类型桥段（桥段不怕老，怕的是表达旧）

## 二、爽点（满分 15 分）

### 检查项

#### 爽点数量与强度（第07讲）

- [ ] **硬约束：≥3 个爽点**，不足 3 个 → 该维度 ≤6 分
- [ ] 爽点类型多样（不全是同一类型）
- [ ] 爽点强度达标（有明确的情绪释放，非轻描淡写）

#### 题材分化（硬约束适配）

爽点的定义随题材变化，评分时需按题材标准：
- **爽感型**（战神/逆袭/复仇）：身份碾压、打脸、逆袭翻盘为核心爽点
- **情感型**（甜宠/日常/治愈）：心动瞬间、误会冰释、双向奔赴、守护行为也算爽点
- **悬疑型**（推理/反转）：真相揭露、反转震撼、逻辑闭环也算爽点

不能用爽感型标准评判甜宠剧本的爽点数量。

#### 四大爆点类型（第07讲）

- [ ] 逆袭成功：小人物崛起有合理理由（奇遇/金手指），非凭空
- [ ] 反转打脸：「意料之外，情理之中」——前期有伏笔铺垫
- [ ] 意外收获：有因果铺垫，无悬浮感（凭空获得）
- [ ] CP 甜宠：男频侧重保护宠爱，女频强调双向奔赴

#### 期待管理（第06讲）

- [ ] 大期待（终极目标）：最晚第二集直白展现，具体化（「三个月赚一千万」优于「成为有钱人」）
- [ ] 小期待（阶段性任务）：大目标拆解为关卡系统，达成时观众感知进步
- [ ] 关键节点反复强化期待

#### 悬念设计（第06讲）

- [ ] 使用至少 2 种信息差类型组合（见下方四模型）
- [ ] 悬念无逻辑硬伤，与剧情紧密结合
- [ ] 无强行反转

**四大信息差模型（自检时逐一对照）：**
1. **主角知，观众和配角不知** — 侦探解谜型（「主角何时揭晓？真相多震撼？」）
2. **配角和观众知，主角不知** — 身份觉醒型（利用观众先知优越感）
3. **观众知，主角和配角不知** — 穿越/年代文型（需注意勿过度降智）
4. **主角和观众知，配角不知** — 最经典最常用：隐藏大佬设定，打脸情节最大化

#### 痛点铺垫（第07讲）

- [ ] 痛点具象化：「尊严被践踏」→「酒席上被泼酒」
- [ ] 男频痛点对齐：安全感/尊严/保护家人
- [ ] 女频痛点对齐：情感安全感/自我价值

#### 困境情节三原则（第07讲）

- [ ] 时长简洁：不长期停留低谷
- [ ] 留希望：暗示反击准备
- [ ] 针对性强：来自清晰对立角色

### 分数锚点

| 分数 | 描述 |
|------|------|
| 3 | 爽点 <3 个或全无；痛点未铺垫直接爽；期待模糊（观众不知主角要什么）；反转凭空无伏笔 |
| 6 | 有 3 个爽点但类型单一（全是打脸）；大期待有但不具体；痛点有但未具象化；困境停留过长 |
| 9 | 3+ 爽点且至少 2 种类型；大期待具体清晰+小期待关卡化；痛点具象+爽点释放形成完整弧线 |
| 12 | 爽点类型丰富且层层递进；期待管理精准（观众始终知道「距离胜利还有多远」）；痛点-困境-爆发节奏完美 |
| 15 | 在 12 分基础上：每个爽点都有前期铺垫+释放+观众情绪共振；蓄力-释放节奏让人「看了还想看」 |

### 避坑（低分特征，第07讲）

- 情绪自嗨：违背受众核心诉求（男频写男主为爱牺牲被视为愚蠢）
- 逻辑缺失：强行推进，人物行为不合逻辑
- 打脸对象错误：应针对主反派，不可欺凌路人/践踏女性/侮辱公职人员

## 三、台词

### 检查项

#### 功能密度（第03/09讲）

- [ ] 台词承载多重功能：推进剧情+揭示关系+制造冲突+强化人设
- [ ] 推动剧情是第一要务（揭示新信息/制造冲突/做关键决定）
- [ ] 台词是人物性格的外化（同一动作因性格不同说法不同）

#### 精简与口语化（第09讲）

- [ ] 「刀刀见血」无冗余废话
- [ ] **台词长度：逐条扫描，标出超过 2 句的台词并建议拆分**
- [ ] 口语化自然（非书面语）
- [ ] 对白为主力 → 画外音辅助 → 内心独白最少

#### 角色区分度（第09讲）

- [ ] 每个角色有辨识度（换角色说也行 = 失败）
- [ ] 称呼匹配层级关系（上级→小X/直呼名；下级→X总/X老师）
- [ ] 角色-语言风格映射表一致

#### 题材语感（第09讲）

- [ ] 古装/奇幻：用词稍文雅但需易懂
- [ ] 情感题材：重细腻度和情绪感染力
- [ ] 轻喜剧：笑点源于性格和情境，避免堆砌过时梗

#### 质感提升（第09讲）

- [ ] 方言/口头禅/行业术语适度点缀
- [ ] 网络热梗谨慎使用（适度拉近但不滥用）

### 分数锚点

| 分数 | 描述 |
|------|------|
| 2 | 台词拗口冗长，心理描写当台词念；角色说话无区分度；大量书面表达；霸总说「我认为我们需要重新审视这段关系」 |
| 5 | 台词通顺但功能单一（只推剧情不揭关系）；角色有区分但不鲜明；偶有超 2 句长台词 |
| 8 | 台词多功能、精简有力；角色有 DNA（能听出谁在说）；称呼/语感匹配题材；无超长台词 |
| 10 | 台词即人设，金句能被观众截图传播；「删掉 1/3 核心信息还在」；每句都承载多重功能如「刀刀见血」 |

### 自检方法（第09讲）

- 大声朗读检验自然度
- 试着删除 1/3 看核心信息是否还在

### 三大误区（第09讲）

1. 台词拗口篇幅过长 → 简化信息量，用符合身份的口吻
2. 心理描写过多（OS 当台词念）→ 用动作+简短台词替代
3. 台词空洞脱离逻辑 → 通过具体细节让人物逐步理解处境

## 四、格式与可拍性（满分 5 分，门槛型）

> 门槛型维度：达标即可（3-5 分），不达标直接拉低总分（1-2 分）。格式好不代表剧本好，但格式差一定过不了稿。

### 检查项

#### 格式规范（第01讲）

- [ ] 场景头完整：集数标注、时间（夜/日）、场景位置（外/内）、场景编号
- [ ] 人物动作用 △ 标记
- [ ] 语气/停顿有标注
- [ ] 景别标注存在
- [ ] 音乐/特殊效果有提示

#### 可拍性评估（第08讲）

- [ ] 场景数量合理：集中拍摄可行，通过布景微调区分
- [ ] 核心人物 3-5 人，总角色不超 20 人
- [ ] 高潮集新角色 ≤2 个
- [ ] 日常集避免引入新角色
- [ ] 无过度依赖高难度元素（大量动作戏/特效/群演）
- [ ] 高危镜头（水下/高空/爆破）仅在核心卖点时使用
- [ ] 单集字数在 500-800 字范围

### 分数锚点

| 分数 | 描述 |
|------|------|
| 1 | 场景头大量缺失；角色超 20 人；无法执行拍摄 |
| 2 | 场景头不规范；无景别标注；字数严重超标；有不必要的高危镜头 |
| 3 | 格式基本规范但有遗漏（缺音乐提示/景别）；角色略多但可控 |
| 4 | 格式完整规范；核心 3-5 人+总角色合理；场景可集中拍摄 |
| 5 | 格式专业精确如工业标准；场景/角色经济高效；「拿到剧本就能直接开拍」 |

### 关键论述（第08讲）

> 「一个能顺利拍出来的 80 分剧本，价值远大于无法落地的 100 分梦想。」

## 五、主线与连贯性

### 检查项

#### 主线清晰度（第04讲）

- [ ] 灵魂三问通过：一句话讲清（主角+目标+阻碍+亮点）？主角一秒共情？故事为谁而写？
- [ ] 目标具体有情绪张力（「首富男主重生归来向背叛兄弟复仇」而非「男孩重生复仇的故事」）
- [ ] 阶段性目标层层递进（关卡系统）
- [ ] 支线为主线"助攻"，不喧宾夺主
- [ ] 大纲聚焦改变命运的关键事件，非时间流水账

#### 前后一致性

- [ ] 与前后集无矛盾
- [ ] 角色行为一致（无突然降智/性格突变）
- [ ] 伏笔有延续和回收

#### 转变逻辑（realism-checklist.md）

- [ ] 认知转变有具体触发物（看到短信/听到对话/撞见场景，非凭空感觉）
- [ ] 情绪转折有过渡状态（信任→[犹豫/困惑]→翻脸，非直接跳变）
- [ ] 关系呈现生活化（「两家月聚一次餐」而非「多年好友」）

### 分数锚点

| 分数 | 描述 |
|------|------|
| 2 | 主线模糊（无法一句话说清）；前后矛盾明显；角色行为跳跃无逻辑；支线喧宾夺主 |
| 5 | 主线有但不够具体（「想复仇」但未说清如何/对谁）；转变缺中间状态；有 1-2 处小矛盾 |
| 8 | 主线一句话清晰+目标具体；转变有触发物和过渡；支线服务主线；前后一致 |
| 10 | 主线如「脊柱」支撑全集；阶段性目标如关卡层层递进；每个转变都有具体触发+合理过渡；伏笔精准回收 |

### 常见误区（第04讲）

1. 大纲写成流水账 → 应只聚焦改变命运的关键事件
2. 副线喧宾夺主 → 支线为主线助攻
3. 幸存者偏差 → 「新人唯一能把握的就是清晰有力的主线」

## 六、反抽象与镜头化

### 检查项

#### 反抽象（硬约束）

- [ ] **△ 段落无纯情绪词**：扫描全文，逐个标出「她很伤心」「气氛凝重」「他很愤怒」类描述
- [ ] 每处纯情绪词给出物理化替换建议（情绪→具体动作/表情/行为）

**纯情绪词判定边界**：
- ❌ 纯情绪词："她很伤心"、"他很愤怒"、"气氛凝重"、"他很紧张"（无物理表现）
- ✅ 不算纯情绪词："气得脸色铁青"（有物理表现"脸色铁青"）、"杀意凛然地拔刀"（有动作"拔刀"）
- **判断标准**：同一描述中是否包含可被摄影机拍到的物理表现。有 → 不算；无 → 算

**A/B/C 三类追加（v1.15.6 新增）**：纯情绪词之外，场景叙事层还要识别以下 3 类"含物理表现但仍不可拍"的小说腔模式，规则细节见 `quality-rules.md#反抽象-画面可拍性规则-轻约束`：

- **A 抽象情绪物化**（即便含"动作词"仍判 fail）："压迫感从身体里溢出来，像冷空气一样扩散"——"溢出/扩散"是动作词但主体"压迫感"不可拍；"空气已经绷紧了"——"绷紧"是动作词但主体"空气"不可拍
- **B 通感/多层比喻**（场景叙事层即便比喻喻体可感仍扣分）："沉闷，像心跳"——拍摄 / 音效制作无法对应"心跳"节奏；"像水结成冰的过程"——眼神变化无法对应"水结冰"视觉
- **C 不可见内心画面**（即便含物理喻体仍判 fail）："胸腔里某根看不见的线被拉紧"——"胸腔里"指向人体内部不可摄；"看不见的 X" 句式直接命中
- **旧判定的边界修正**：现行"含物理表现就放过"的判据只覆盖"纯情绪词"类。对 A/B/C 三类，即便句中含物理表现或动作词，**仍判 fail**
- **与维度 7 AI Slop 分工（范围冲突 C3 处理）**：场景叙事层 + OS 超阈的 A/B/C 违规归本维度扣分；对白层 + VO 超阈归维度 7；**单集场景叙事层比喻句堆叠 ≥4 处**（即便单句都过 3 问）归维度 7 -2 分，本维度不重复扣
- **与"无上帝视角"边界澄清（C6 处理）**：C 类 = 单句人体内部不可摄画面；上帝视角 = 跨场景叙述剧中人不可知之事。同一句不重复扣分

#### 镜头化思维（第08/10讲）

- [ ] 少写心理多写行为——通过表情/动作/沉默传达情绪
- [ ] 关键信息通过剧情/对白/行为呈现，不依赖观众脑补（无上帝视角）
- [ ] 大段静态描写已转化为外化动作与精炼对话
- [ ] 「简单直给」：开场十几秒内让观众理解看点

### 分数锚点

| 分数 | 描述 |
|------|------|
| 2 | 大量纯情绪词（≥5 处）；依赖心理描写推进故事；关键信息靠观众脑补；有大段静态描写 |
| 5 | 有 2-3 处纯情绪词；部分场景用心理描写代替行为；偶有上帝视角 |
| 8 | 全文无纯情绪词（或仅 1 处已标出）；情绪通过物理动作传达；信息通过剧情/对白呈现 |
| 10 | 每个情绪点都有画面感的物理表达；「读每句话脑中能浮现具体镜头」；零心理描写零上帝视角；**且无 A/B/C 类残留（抽象物化 / 通感比喻 / 不可见内心）** |

## 七、AI Slop

### 检查项

#### 台词书面化

- [ ] 扫描全部台词，标出书面表达：「我认为我们应该」「在某种程度上」「不可否认」「事实上」「从某个角度来说」
- [ ] 每处给出口语化替换

#### 情绪转变过平滑

- [ ] 检查角色情绪/立场转变是否缺少抗拒、犹豫、反复等中间状态
- [ ] 「她听完立刻原谅了他」= AI Slop，需要加入挣扎过程

#### 巧合堆砌

- [ ] 统计「正好」「恰好」「刚好」「碰巧」「没想到」出现次数
- [ ] **全集 ≥3 次 → 判定巧合过多**，每处标出并建议用因果逻辑替代

#### AI 生成痕迹（补充）

- [ ] 无过度完美的角色（全面优秀无缺点）
- [ ] 无过度对称的情节结构（机械化的三段式）
- [ ] 无「总结性」结尾台词（角色像在做 presentation）

#### 场景叙事层比喻堆叠扫描（v1.15.6 新增，与维度 6 分工）

**扫描**：统计全集**场景叙事层**（△ / 镜头号 / `（BGM：...）` / `（音效：...）` / `[音乐]`（老格式兼容）/ 闪回段落开头的描述）中含比喻的句子数量（"像 X" / "仿佛 X" / "如 X 一般" 等比喻句式）。

**阈值**：
- ≤3 处 → 不扣分（少量比喻可增强画面质感）
- ≥4 处 → **-2 分**（堆叠成灾，属于 AI 小说腔特征——单句过维度 6 的 3 问仍可能堆叠）

**与维度 6 分工（范围冲突 C3 处理）**：
- 维度 6：场景叙事层 + OS 超阈的 A/B/C 类单句违规（每处 -1，上限 -3）
- 维度 7 本项：**单集场景叙事层比喻句数量堆叠**（不管单句过不过 3 问，数量达标就扣）
- 同一句不重复扣分：如一句同时命中维度 6 的 B 类比喻，**优先归维度 6**，不再计入本项数量统计

**VO 超阈也归本维度**：每集 VO 总字数 > 本集总字数 20% **-1 分**；单段 VO > 3 句 **-1 分**（避免模型把小说腔全塞 VO 规避维度 6）

### 分数锚点

| 分数 | 描述 |
|------|------|
| 2 | 书面表达 ≥5 处；情绪转变全部一步到位无中间态；巧合词 ≥5 次；明显 AI 生成质感 |
| 5 | 书面表达 2-3 处；有 1-2 处情绪跳变；巧合词 2-3 次；偶有 AI 痕迹 |
| 8 | 书面表达 ≤1 处；情绪转变有过渡；巧合词 ≤1 次；读起来像人写的 |
| 10 | 零书面化、零巧合堆砌；情绪转变层次丰富（抗拒→犹豫→松动→接受）；完全无 AI 痕迹 |

## 八、考据可追溯性（满分 10 分，题材分化）

> 本维度仅对厚型/中型题材计入总分；轻型题材记 N/A，总分按 70 计算。题材分类见 `genre-guide.md#考据强度判定`。

### 检查项

#### 专业细节映射 bible

- [ ] 所有专业术语 / 官名 / 制度 / 数字 / 药物剂量 / 法条编号都能映射到 setting-bible.md 一条
- [ ] 引用 `[待人工核源]` 字段时加 `[考据待定]` 标注
- [ ] 引用 `[来源冲突]` 字段 = 0（应在 /考据 阶段已修复）

#### 时代/领域常识无穿帮

- [ ] 称谓符合时代规范（如南齐用「郎君」非「公子」）
- [ ] 饮食/服饰/工具/物价符合时代
- [ ] 现代题材职业行为符合行业实际（医疗操作流程 / 法律程序 / 金融工具使用）

#### 制度规则使用一致

- [ ] 同一规则前后集应用一致（不出现"九品中正"前后规则不同）
- [ ] 大事件处置遵循 bible「大事件处置补丁」字段

#### 虚构白名单合规

- [ ] 虚构成分都列入 setting-bible.md 虚构白名单
- [ ] 虚构地名/药物/制度未与真实事实冲突

### 雷区否决项（修订 8）

雷区是辨别力最强的检查项，单独命中规则：

- **命中 1 条雷区** → 本维度上限 6 分
- **命中 ≥ 2 条雷区** → 本维度上限 3 分
- **关键事实编造（与权威源直接矛盾）** → ≤ 3 分

雷区分类参考 `research-guide.md#反模式`：精确年份 / 具体官职品级 / 药物剂量 / 抢救参数 / 法条编号 / 监管细则。

### 硬约束

- **bible 不存在但题材为厚型** → 本维度 0 分
- **bible 状态 = locked 但剧本含 [虚构] 标注** → 0 分
- **题材为轻型** → N/A，不计入总分

### 编造检测启发式

剧本中出现以下模式，自动检索 bible 验证，无映射的视为编造：

- 形如「{朝代}{N}年{月}月{日}日」的精确日期
- 形如「{品级}{官名}」（如「从五品下{官名}」）
- 形如「{数字}{单位}{药物名}」（如「5mg地塞米松」）
- 形如「{数字}J双相波」「{数字}次/分」（医疗参数）
- 形如「刑法第{N}条第{M}款」「证监会{年份}年第{N}号公告」

### 分数锚点

| 分数 | 描述 |
|------|------|
| 0 | 厚型题材无 bible / locked 状态有 [虚构] 标注 / 与权威源直接矛盾 |
| 2 | 命中 ≥ 2 条雷区 / 多处编造未映射 |
| 4 | 命中 1 条雷区 / 1-2 处常识穿帮 |
| 6 | 大部分细节有映射但少量 `[考据待定]` 未在 bible 中 / 1 处轻微穿帮 |
| 8 | 全部专业细节有 bible 映射 / 时代常识无穿帮 / 虚构白名单合规 |
| 10 | 8 分基础上：bible 主动引用率高（细节饱满）/ 虚构成分有合理框架 / 制度规则推动剧情 |

## 评分标准与平台过稿预估

总分动态——**厚型/中型 满分 80**（含第 8 维度），**轻型满分 70**（第 8 维度 N/A 不计入）。阈值按 80%/60%/40% 比例分档：

| 满分 | 优秀 ✅ | 需改 ⚠️ | 困难 ❌ | 重写 🚫（导出阻断）|
|------|--------|--------|--------|------------------|
| **80（厚型/中型）**| 64-80 | 48-63 | 32-47 | <32 |
| **70（轻型）**| 56-70 | 42-55 | 28-41 | <28 |

> **历史说明**：v1.10.0 及以前文档标"总分 75"为历史误标（实际 7 维度分值和 = 10+15+10+5+10+10+10 = 70）。本版校正为真实总分。

> 注意：过稿预估基于行业编剧方法论标准，非平台官方审核算法。实际过稿还受题材热度、平台当期需求、制作方资质等因素影响。
