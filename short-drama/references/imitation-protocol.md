# 仿写方法论 — 爆款解构·迁移·热点融入

> 供 `/仿写` 命令执行时加载。三阶段管线：骨架提取 → 题材迁移 + 时代底色 → 逐集填表仿写。

---

## Phase 1：骨架提取

### 输入处理

| 输入形式 | 处理方式 |
|---------|---------|
| 用户粘贴文本 | 直接进入提取 |
| 提供 `.md` 文件路径 | Read 文件，进入提取 |
| 提供 `.docx` 文件路径 | `python3 -c "import docx; doc=docx.Document('PATH'); [print(p.text) for p in doc.paragraphs if p.text.strip()]"` 提取正文 |
| 只给剧名/描述 | 用模型知识重建骨架，所有字段标 `[推断]`，提醒用户准确度有限 |

> **大文件警告**：超过 50 集的源剧正文不要全读。告知用户"只需提供前 15 集和全剧大纲摘要（每集一句话）"，足以提取结构 DNA。

### 六维结构 DNA

提取时严格按以下 6 个维度，不自由发挥：

**维度 1：身份密码**
- 主角隐藏了什么身份/能力/信息？对立方不知道什么？
- 具体形式（举例）：真实身份 / 背景关系 / 专业能力 / 情报优势
- 身份何时首次泄露？何时全揭？

**维度 2：压迫机制**
- 什么力量以什么方式持续打压主角？
- 压迫者的权力来源（家族权威 / 职场权力 / 社会地位 / 情感控制 / 经济依附）
- 压迫的边界：压迫者"不知道的那件事"是什么？

**维度 3：情感引爆点分布**
- 全剧共几个主要情感引爆点（定义：让观众停不下来必须看下集的情绪峰值）
- 每个引爆点的类型：背叛揭露 / 权力反转 / 牺牲 / 重逢 / 清算 / 告白
- 引爆点在集数分布中的位置（约在第几集、占全剧 %）

**维度 4：爽点密度 & 类型**
- 每 5 集区间有几个爽点？（节奏密度）
- 主要爽点类型：身份暴露型 / 打脸型 / 逆袭型 / 助力出现型 / 知情者反应型
- 哪类爽点最集中在哪个阶段？

**维度 5：付费卡点设计模式**
- 在哪些节点设置了付费墙？
- 卡点前的叙事技术：悬念留白 / 高潮截断 / 信息差扩大 / 情感顶点前停帧
- 每个卡点的"诱饵情绪"是什么（好奇 / 愤怒 / 心疼 / 期待）？

**维度 6：节奏曲线**
- 压迫期：占全剧几成？节奏特征？
- 反转蓄力期：从第几集开始主角蓄力？外部助力何时出现？
- 清算爆发期：主要清算集中在哪个区间？用什么手段清算？
- 结局类型：硬爽 / 开放 / 糖结局？有无 EP1 钩子（结局倒叙开场）？

### 输出格式

写入 `~/short-drama-projects/<项目>/imitation/source-skeleton.md`：

```markdown
# 源剧骨架 — [参考剧名/描述]
提取日期：YYYY-MM-DD
数据来源：[粘贴文本 / 文件路径 / 推断]（推断的标注"[推断]"）

## 身份密码
...

## 压迫机制
...

## 情感引爆点分布
| 引爆点 | 类型 | 约在集数 | 占全剧% |
|-------|------|---------|--------|
| ...   | ...  | ...     | ...    |

## 爽点密度 & 类型
...

## 付费卡点设计模式
...

## 节奏曲线
- 压迫期：第 1-XX 集（XX%）…
- 反转蓄力期：第 XX 集起…
- 清算爆发期：第 XX-XX 集…
- 结局类型：…
```

## Phase 2：题材迁移 + 时代底色 + 热点研究

### Step 2.1：变体方向生成

基于 source-skeleton.md，生成 3 个变体方向。**结构逻辑不变，替换题材皮**：

- 保留：骨架六维中的逻辑结构（压迫来源类型、身份密码形式、引爆点密度）
- 替换：场景（时代/地点/行业）、人物弧线的具体化身、冲突的社会载体

每个变体用一行描述：

```
变体 A：[赛道方向] — 压迫机制迁移为[X]，身份密码变体为[Y]，核心张力变为[Z]
变体 B：...
变体 C：...
```

### Step 2.2：用户选择变体

用户回复变体编号，或提出调整方向。

### Step 2.3：时代底色推导（LLM 推导，用户确认）

> **原则**：时代底色从故事 DNA 里长出来，不是让用户凭空填写。  
> LLM 读 source-skeleton.md + 用户选定变体 → 推导 2-3 个候选社会母体。

**推导逻辑**：
1. 看压迫机制的权力来源 → 映射当代集体焦虑
2. 看身份密码的隐藏方式 → 映射社会中哪类人需要"隐藏自己"才能存活
3. 看清算手段 → 映射什么价值观在观众心理中有"正义"投射

输出格式：

```
基于源剧骨架（[一句话骨架摘要]）+ [变体方向]，这部剧的时代底色候选：

A) [社会母体 A]——[一句话解释为什么骨架指向这个母体]
B) [社会母体 B]——…
C) [社会母体 C]——…

请确认一个，或告诉我你想要的底色方向。
```

确认后，时代底色写入 new-setting.md `时代底色` 字段，作为后续热点筛选和 Phase 3 填表的全局约束。

### Step 2.4：热点研究

基于确认的时代底色，**并行发起 3 类 WebSearch**：

| 搜索类别 | 目的 | 参考关键词构造 |
|---------|------|-------------|
| 类别 1：社会话题 | 找与母体共振的社会议题/新闻事件 | `时代底色关键词 + 热点 OR 话题 OR 引发讨论 site:weibo.com OR zhihu.com` |
| 类别 2：情感共鸣 | 找大众情绪出口（委屈/愤怒/期待被看见） | `时代底色关键词 + 情感 OR 扎心 OR "说出了多少人的心声"` |
| 类别 3：网络热梗 | 找可嵌入剧情的流行语/近期事件 | `YYYY年 + 赛道关键词 + 爆款 OR 流行 OR 热搜` |

**搜索结果处理**：每类精选 3-5 条，列出条目 + 一句话为什么与本剧相关，让用户选择 2-3 条融入。

> **融入层次原则**（避免表面贴标签）：
>
> - **优先：世界观层** — 热点塑造故事世界的"当代背景噪音"（角色的行为动机来自这个世界的规则）
> - **其次：人物动机层** — 热点直接驱动某个角色的选择（而不是只被提到）
> - **最后才是：剧情事件层** — 热点作为一个具体发生的事件出现
>
> **禁止**：热点作为台词里的名词标签出现但与剧情逻辑无关。

### Step 2.5：写入 new-setting.md

写入 `~/short-drama-projects/<项目>/imitation/new-setting.md`：

```markdown
# 迁移设定 — [新剧名（暂定）]
生成日期：YYYY-MM-DD

## 题材赛道
[变体方向一句话]

## 时代底色
[确认的社会母体] — [一句话说明这个底色如何渗入故事逻辑]

## 融入热点
| 热点 | 融入层次 | 具体融入方式 |
|-----|---------|-----------|
| [热点1] | 世界观层 / 人物动机层 / 剧情事件层 | … |
| [热点2] | … | … |

## 骨架迁移映射
| 源剧骨架维度 | 源剧写法 | 本剧迁移写法 |
|------------|---------|-----------|
| 身份密码 | … | … |
| 压迫机制 | … | … |
| 核心引爆点类型 | … | … |
| 爽点主力类型 | … | … |
| 付费卡点诱饵情绪 | … | … |
| 结局类型 | … | … |
```

## Phase 3：逐集填表仿写

### 执行原则

1. **只读 source-skeleton.md + new-setting.md**，不重读源剧正文——防内容污染，强制从结构逻辑生成
2. 每集先生成**对齐表**，再从对齐表填写正文——对齐表是防结构漂移的硬约束
3. 输出文件与 `/分集` 完全一致：`~/short-drama-projects/<项目>/episodes/ep{NNN}.md`，格式遵守 `references/format-control.md`
4. 批量执行：用户可指定集数范围（如 `/仿写 第5-8集`）；单次默认生成 1 集

### 每集生成流程

**Step A：生成对齐表**

```
## ep{NNN} 对齐表

| 结构要素 | 源剧模式（来自 source-skeleton） | 本集迁移写法 |
|---------|-------------------------------|-----------|
| 情绪触发点 | [源剧该位置的情绪类型] | [新剧同位置的情绪事件] |
| 冲突核心 | [源剧压迫形式] | [迁移后的压迫具体化] |
| 爽点类型 | [有/无 + 类型] | [对应爽点的新形式] |
| 热点融入 | N/A | [本集融入哪条热点、以何方式] |
| 付费卡点 | [有/无 + 诱饵情绪] | [本集卡点设计] |
| 台词风格锚 | [源剧典型台词风格] | [本集台词方向：口语化程度/情绪密度] |
```

**Step B：从对齐表生成正文**

- 严格按对齐表中"本集迁移写法"列写正文，不偏离
- 热点融入必须落到对齐表指定的层次（世界观/动机/事件），不得降格为台词标签
- 集末 hooks 对应 source-skeleton 中的节奏曲线——处于压迫期的集数 hook 要强化信息差，处于清算期的集数 hook 可做满足感铺垫

**Step C：写文件 + 提示**

写完正文后：
- 写入 `~/short-drama-projects/<项目>/episodes/ep{NNN}.md`
- 更新 `.drama-state.json#currentEpisode` 和 `#completedEpisodes`
- 本集末尾提示：

```
[仿写完成 ep{NNN}]
对齐表已锁定结构，正文从表中生成。

→ 继续写：/仿写 {N+1}
→ 检验质量：/自检 {N}（多维打分，确认结构对齐度）
→ 降 AI 味：/人化（建议在全稿写完后批量处理）
→ 分镜：/分镜 {N}
```
