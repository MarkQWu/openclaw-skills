# 仿写方法论 — 爆款解构·迁移·热点融入

> 供 `/仿写` 命令执行时加载。三阶段管线：骨架提取 → 题材迁移 + 时代底色 → 逐集填表仿写。

---

## Phase 1：骨架提取

### 输入处理

| 输入形式 | 处理方式 |
|---------|---------|
| 用户粘贴文本 | 写入 `imitation/source-script.md`（原文落盘），再进入提取 |
| 提供 `.md` 文件路径 | Read 文件 → cp 到 `imitation/source-script.md`（原文落盘），再进入提取 |
| 提供 `.docx` 文件路径 | `python3 -c "import docx; doc=docx.Document('PATH'); [print(p.text) for p in doc.paragraphs if p.text.strip()]"` 提取正文 → 写入 `imitation/source-script.md`（原文落盘），再进入提取 |
| 只给剧名/描述 | 用模型知识重建骨架，所有字段标 `[推断]`，提醒用户准确度有限。**不生成 source-script.md**（无原文可落盘） |

**原文落盘规则**（有实际文本时必须执行）：
- 目标路径：`~/short-drama-projects/<项目>/imitation/source-script.md`
- 若源文件已是 `.md`：`cp <源路径> <目标路径>`（保留原格式）
- 若是粘贴文本或 `.docx` 提取出的纯文本：Write 写入 `source-script.md`，在文件顶部加注：
  ```
  <!-- 原始来源：[粘贴文本 / 文件名.docx] | 提取日期：YYYY-MM-DD -->
  ```
- 落盘后在 Phase 1 提取完成提示里告知用户："原文已保存至 `imitation/source-script.md`，提取骨架见 `imitation/source-skeleton.md`"

> **大文件自动压缩**：超过 50 集的源剧，优先调用 `references/condense-source.py` 自动生成 `imitation/source-script-condensed.md`（前10集全文 + 其余结构摘要，节省约 65% token）。脚本不可用时，要求用户提供：前10集 + 其余每集按格式「Ep{N}: 核心冲突 | 节奏段:[压迫/蓄力/清算] | 爽点:[类型/无] | 炸点:[类型/无] | 卡点:[有/无]」。落盘时写压缩后文件路径。

### 六维结构 DNA

提取时严格按以下 6 个维度，不自由发挥：

**维度 1：身份密码**
- 主角隐藏了什么身份/能力/信息？对立方不知道什么？
- 具体形式（举例）：真实身份 / 背景关系 / 专业能力 / 情报优势
- 身份何时首次泄露？何时全揭？

**维度 2：压迫机制**
- 什么力量以什么方式持续打压主角？
- 压迫者的权力来源（家族权威 / 职场权力 / 社会地位 / 情感控制 / 经济依附）
- 压迫的边界：压迫者"不知道的那件事"是什么？

**维度 3：情感引爆点分布**
- 全剧共几个主要情感引爆点（定义：让观众停不下来必须看下集的情绪峰值）
- 每个引爆点的类型：背叛揭露 / 权力反转 / 牺牲 / 重逢 / 清算 / 告白
- 引爆点在集数分布中的位置（约在第几集、占全剧 %）

**维度 4：爽点密度 & 类型**
- 每 5 集区间有几个爽点？（节奏密度）
- 主要爽点类型：身份暴露型 / 打脸型 / 逆袭型 / 助力出现型 / 知情者反应型
- 哪类爽点最集中在哪个阶段？

**维度 5：付费卡点设计模式**
- 在哪些节点设置了付费墙？
- 卡点前的叙事技术：悬念留白 / 高潮截断 / 信息差扩大 / 情感顶点前停帧
- 每个卡点的"诱饵情绪"是什么（好奇 / 愤怒 / 心疼 / 期待）？

**维度 6：节奏曲线**
- 压迫期：占全剧几成？节奏特征？
- 反转蓄力期：从第几集开始主角蓄力？外部助力何时出现？
- 清算爆发期：主要清算集中在哪个区间？用什么手段清算？
- 结局类型：硬爽 / 开放 / 糖结局？有无 EP1 钩子（结局倒叙开场）？

### 输出格式

写入 `~/short-drama-projects/<项目>/imitation/source-skeleton.md`：

```markdown
# 源剧骨架 — [参考剧名/描述]
提取日期：YYYY-MM-DD
数据来源：[粘贴文本 / 文件路径 / 推断]（推断的标注"[推断]"）
原文存档：[imitation/source-script.md（如有）/ 无（仅推断）]

## 身份密码
...

## 压迫机制
...

## 情感引爆点分布
| 引爆点 | 类型 | 约在集数 | 占全剧% |
|-------|------|---------|--------|
| ...   | ...  | ...     | ...    |

## 爽点密度 & 类型
...

## 付费卡点设计模式
...

## 节奏曲线
- 压迫期：第 1-XX 集（XX%）…
- 反转蓄力期：第 XX 集起…
- 清算爆发期：第 XX-XX 集…
- 结局类型：…
```

## Phase 2：题材迁移 + 时代底色 + 热点研究

### Step 2.1：变体方向生成

基于 source-skeleton.md，生成 3 个变体方向。**结构逻辑不变，替换题材皮**：

- 保留：骨架六维中的逻辑结构（压迫来源类型、身份密码形式、引爆点密度）
- 替换：场景（时代/地点/行业）、人物弧线的具体化身、冲突的社会载体

每个变体用一行描述：

```
变体 A：[赛道方向] — 压迫机制迁移为[X]，身份密码变体为[Y]，核心张力变为[Z]
变体 B：...
变体 C：...
```

### Step 2.2：用户选择变体

用户回复变体编号，或提出调整方向。

### Step 2.3：时代底色推导（LLM 推导，用户确认）

> **原则**：时代底色从故事 DNA 里长出来，不是让用户凭空填写。  
> LLM 读 source-skeleton.md + 用户选定变体 → 推导 2-3 个候选社会母体。

**推导逻辑**：
1. 看压迫机制的权力来源 → 映射当代集体焦虑
2. 看身份密码的隐藏方式 → 映射社会中哪类人需要"隐藏自己"才能存活
3. 看清算手段 → 映射什么价值观在观众心理中有"正义"投射

输出格式：

```
基于源剧骨架（[一句话骨架摘要]）+ [变体方向]，这部剧的时代底色候选：

A) [社会母体 A]——[一句话解释为什么骨架指向这个母体]
B) [社会母体 B]——…
C) [社会母体 C]——…

请确认一个，或告诉我你想要的底色方向。
```

确认后，时代底色写入 new-setting.md `时代底色` 字段，作为后续热点筛选和 Phase 3 填表的全局约束。

### Step 2.3.5：目标平台推导（LLM 推导，用户确认）

基于确认的变体方向 + 时代底色，推导最匹配的发行平台：

| 平台特征 | 匹配信号 |
|---------|---------|
| 抖音 | 都市情感/职场/爽感密集，节奏≤90s/集，付费墙早 |
| 快手 | 家庭伦理/萌宝/农村逆袭，共情浓，IAA 为主 |
| 红果 | 精品长集，IP 翻拍/古装，投拍级画质 |
| 出海（ReelShort/DramaBox）| mafia/werewolf/dark romance，英文，付费墙 ep5 前 |

输出格式（一句话）：

```
基于[变体方向]+[时代底色]，目标平台推荐：[平台名] — [一句话节奏/格式约束]
（如需切换到出海模式请在确认后输入 /出海）
请确认，或指定其他平台。
```

确认后写入 new-setting.md `目标平台` 字段，约束 Phase 3 的集长/节奏/付费墙位置。

### Step 2.4：热点研究

基于确认的时代底色，**并行发起 3 类 WebSearch**：

| 搜索类别 | 目的 | 参考关键词构造 |
|---------|------|-------------|
| 类别 1：社会话题 | 找与母体共振的社会议题/新闻事件 | `时代底色关键词 + 热点 OR 话题 OR 引发讨论 site:weibo.com OR zhihu.com` |
| 类别 2：情感共鸣 | 找大众情绪出口（委屈/愤怒/期待被看见） | `时代底色关键词 + 情感 OR 扎心 OR "说出了多少人的心声"` |
| 类别 3：网络热梗 | 找可嵌入剧情的流行语/近期事件 | `YYYY年 + 赛道关键词 + 爆款 OR 流行 OR 热搜` |

**搜索结果处理**：每类精选 3-5 条，列出条目 + 一句话为什么与本剧相关，让用户选择 2-3 条融入。

> **融入层次原则**（避免表面贴标签）：
>
> - **优先：世界观层** — 热点塑造故事世界的"当代背景噪音"（角色的行为动机来自这个世界的规则）
> - **其次：人物动机层** — 热点直接驱动某个角色的选择（而不是只被提到）
> - **最后才是：剧情事件层** — 热点作为一个具体发生的事件出现
>
> **禁止**：热点作为台词里的名词标签出现但与剧情逻辑无关。

### Step 2.4.5：综合推导（LLM 自动完成，无需用户输入）

用户选定热点后，LLM 综合 source-skeleton.md + new-setting（变体/母题/平台/热点）自动推导以下 4 项，**一次性输出供用户确认**：

**① 核心人物焕新**（从骨架身份密码 + 压迫机制 + 已选变体推导）

推导原则：
- 主角：从身份密码维度提取"隐藏能力/身份"→ 在新题材中找对应职业/阶层/社会角色
- 反派：从压迫机制权力来源 → 在新题材中找对应的权力主体
- 情感对象：从情感引爆点的"情感纠缠方"→ 在新题材中找对应关系
- 恶毒配角：从压迫机制的"帮凶/催化剂"→ 在新题材中具象化（例：婆婆/小姑/塑料姐妹/踩低同事/妒忌亲戚，与反派区别在于"不掌主要权力，专门添堵"）

每个角色必须有**独特动作锚**（视觉可识别的习惯动作 / 口头禅），用于 Phase 3 对齐表「人物动作锚」行的数据源。

**② 关键场景/道具映射**（从 source-skeleton 的爽点密度 + 情感引爆点推导）

找原剧中承载高情绪密度的 6-10 个标志性场面/道具 → 在新题材中找等价的视觉载体。

原则：道具/场面要「可拍」（成本合理）且「情绪等价」（能承载相同情绪强度）。

**③ 剧名候选 + Logline**（综合变体 + 母题 + 角色 + 目标平台生成）

剧名：给出 2 个候选（A 偏爽感/B 偏情感），命名规律参考目标平台爆款（抖音喜欢「我是XXX」「CEO/总裁」直白挂牌；快手偏「妈妈/孩子」情感词；出海偏「The Alpha's/Mafia Lord's」所有格）。

Logline 格式：「[主角身份] + [触发事件] → [核心关系/对抗] + [情绪基调]」，一句话，不超 40 字（中文）/ 25 词（英文）。

**④ 预判切片爆点**（从 source-skeleton 的引爆点分布 + 节奏曲线推导）

基于源剧情感引爆点 → 在新剧相同节奏位置预判 3-5 个「最可能成为短视频切片」的高光时刻，每条附：
- 爆点描述（一句话，含场景/情绪/角色动作）
- 大致集数区间（第 X-Y 集）
- 切片类型：打脸型 / 揭秘型 / 反转型 / 告白型 / 清算型

输出为一次性确认清单，用户可直接回复「确认」或修改某项，之后写入 new-setting.md。

## Phase 3：逐集填表仿写

### 执行原则

1. **只读 source-skeleton.md + new-setting.md**，不重读源剧正文——防内容污染，强制从结构逻辑生成
2. 每集先生成**对齐表**，再从对齐表填写正文——对齐表是防结构漂移的硬约束
3. 输出文件与 `/分集` 完全一致：`~/short-drama-projects/<项目>/episodes/ep{NNN}.md`，格式遵守 `references/format-control.md`
4. 批量执行：用户可指定集数范围（如 `/仿写 第5-8集`）；单次默认生成 1 集

### 每集生成流程

**Step A：生成对齐表**

```
## ep{NNN} 对齐表

| 结构要素 | 源剧模式（来自 source-skeleton） | 本集迁移写法 |
|---------|-------------------------------|-----------|
| 情绪触发点 | [源剧该位置的情绪类型] | [新剧同位置的情绪事件] |
| 冲突核心 | [源剧压迫形式] | [迁移后的压迫具体化] |
| 爽点类型 | [有/无 + 类型] | [对应爽点的新形式] |
| 热点融入 | N/A | [本集融入哪条热点、以何方式] |
| 付费卡点 | [有/无 + 诱饵情绪] | [本集卡点设计] |
| 台词风格锚 | [源剧典型台词风格] | [本集台词方向：对照 new-setting.md 核心人物焕新表的「独特动作/口头禅」] |
| 人物动作锚 | [源剧对应角色的标志性动作] | [本集哪个角色、用 new-setting.md 中该角色的独特动作/口头禅、在哪个场次触发] |
| 切片承载 | N/A | [本集是否承载「预判切片爆点」中的某项；是→注明切片编号+场次位置；否→「铺垫/过渡集」] |
```

**Step B：从对齐表生成正文**

- 严格按对齐表中"本集迁移写法"列写正文，不偏离
- 热点融入必须落到对齐表指定的层次（世界观/动机/事件），不得降格为台词标签
- 集末 hooks 对应 source-skeleton 中的节奏曲线——处于压迫期的集数 hook 要强化信息差，处于清算期的集数 hook 可做满足感铺垫

**Step C：写文件 + 提示**

写完正文后：
- 写入 `~/short-drama-projects/<项目>/episodes/ep{NNN}.md`
- 按 Read-Modify-Write 协议追加本集到 `.drama-state.json#completedEpisodes`
- 本集末尾提示：

```
[仿写完成 ep{NNN}]
对齐表已锁定结构，正文从表中生成。

→ 继续写：/仿写 {N+1}
→ 检验质量：/自检 {N}（多维打分，确认结构对齐度）
→ 分镜：/分镜 {N}
```
