# 已用台词库协议（used-lines-protocol）

跨集台词去重机制的完整规范。被 `SKILL.md` `/分集` 和 `/自检` 命令引用、被 `quality-rules.md#已用台词库跨集去重机制` 引用。

## 为什么需要这个机制

AI 写多集剧本时有一个已知系统性问题：**锁死一个生动表达反复复读**。典型表现：

- 某位已故角色的"遗言"在多集中被不同角色重复引用
- 煽情金句（如「如果有来生我还要遇见你」）在多集中反复出现 3 次以上
- 高频礼貌套语（如「请多指教」）密集堆积在少数几集

根因：现有 `#上下文连贯性` 只是定性提醒"回顾前面集数"，没有结构化扫描机制。`used-lines.md` 是解法——给 Claude 一份**已用台词清单**作为正向召回锚点，生成时自动从候选集剔除。

**框架关键**：这不是"黑名单"（否定约束，LLM 对否定遵守率低），是**"已用清单"（正向召回，模型从候选集剔除）**。措辞差别在实际输出质量上不可忽略。

## 格式规范

```markdown
# 已用台词库

> 本文件记录已写入剧本的高复读风险台词，供后续集数生成时回避重复。
> 由 /分集 命令每集自动追加。禁止 overwrite，只追加。

## ep001

- "执念太深，终究难逃此劫" [老夫人][灵堂临终][遗言锚句]
- "玉佩不能碎" [陆离][赶来路上][口头禅/执念锚句]

## ep002

- "这世上哪有什么巧合" [沈清辞][书房对峙][宿命情绪]
- "三寸三分，不差一毫" [陆离][验尸现场][专业术语+标志性短语]

...
```

**每条格式**：`- "台词原文" [角色][场景 one-liner][类别]`

- 台词原文：必须精确引号包裹，保留标点，不简写不改写
- 角色：与 `characters.md` 中的角色名完全一致（不用别名/昵称）
- 场景 one-liner：5-10 字描述场景（便于用户扫读时联想上下文）
- 类别（共 9 类，v1.20.0 新增第 9 类「首集传播句」）：从下列枚举选，多选用 `/` 分隔
  - `情绪锚句`：带强情感色彩的短句（"我不甘心"）
  - `遗言锚句`：临终/告别/绝笔类
  - `宿命情绪`：命运/执念/因果类
  - `口头禅`：角色标志性重复用语
  - `誓言承诺`：诺言/盟誓（"我发誓..."）
  - `专业术语`：带题材印记的短语（仵作行话、官场隐语等）
  - `标志性短语`：3 字以上独特组合（"三寸三分，不差一毫"）
  - `特殊意象`：带具体意象的比喻（"像一滴融进海里的血"）
  - `首集传播句`：**第 1 集必出 1 条**——偏短（≤12 字优先，上限 20 字）/ 角色化（说话人 ID 强，不写 AI 通用金句腔）/ 可回收（后续应被打脸或呼应）。详见 §首集传播句

## 写入规则（/分集 命令在每集写完后执行）

### 判定标准：什么该追加

提取标准：**3-5 条/集**。**硬要求下限 3 条**——少于 3 条说明没认真扫，不达标。

应追加：
- 情绪锚句、遗言、誓言、口头禅、特殊意象
- 3 字以上标志性形容词组合
- 重点概念短语（和本剧世界观/人设强绑定的表达）
- 反复出现在同一集中的短句（已经被本集用作节拍器的）

**不**追加：
- 功能性台词：「走吧」「来了」「知道了」「好」
- 信息传递台词：「今天是初九」「他在后院」
- 场景切换提示：「我们出发吧」
- 纯礼貌套语：「谢谢」「不客气」（除非角色化明显如陆离的「请多指教」）

**反向启发式（选不准时的判定法）**：假设这句台词放到 ep10 重说一次，观众会记得 ep2 说过吗？
- 会记得 → **必追加**（说明够独特，值得防复用）
- 不会记得 → **不追加**（说明是通用表达，不占黑名单空间）

这一条可作为 3-5 条硬下限下的"补位"判断——当已选出 2 条明确的、第 3 条在多个候选之间犹豫时用。

### 追加协议（强制 Read-Modify-Write）

1. **Read `used-lines.md`**（不存在则视为空文件，后续 Write 创建）
2. **追加新 section**：`## epNNN`（三位数补零）+ 该集 3-5 条台词
3. **Write 完整内容**（不 overwrite 已有 section）
4. **不得**：直接 Write 覆盖整个文件、跳过 Read 步骤、少于 3 条敷衍

## 读取规则（/分集 命令在生成前执行）

### 加载策略（避免上下文爆炸）

生成第 N 集前的 Read 策略按 N 值分档：

| 已有集数 | Read 策略 |
|---|---|
| N ≤ 10 | 全量 Read，所有 section 全部展示给 Claude |
| 10 < N ≤ 30 | 最近 10 集全量 + 更早集数仅展示 `## epNNN` 标题行 + 每集前 2 条 |
| N > 30 | 最近 10 集全量 + 更早集数仅标题行 + 每集前 1 条 + 提示用户"used-lines.md 已较长，可手动整理高频项" |

### 使用语境

Claude 在生成第 N 集时必须：

1. 把加载的 used-lines.md 内容视为"已用清单，本集生成时从候选集剔除"
2. 遇到要写情绪锚句/遗言/口头禅等类别时，扫一眼已用清单，优先选新的表达
3. 允许复用的例外：
   - **角色口头禅**（同一角色在不同集表现一致性需要重复）——但同一口头禅不应在 3 集内重复
   - **世界观专业术语**（仵作行话、修仙术语等固定表达，必然重复）
   - **明确的意象回响**（剧情刻意设置的 callback，如 ep001 开场句在 ep050 大结局回响）

## /自检 扫描规则

`/自检` 第 5 维度"连贯性"新增子项："**跨集台词复用扫描**"：

### 扫描算法

1. 读取本集 `episodes/epNNN.md`
2. 读取历史 `used-lines.md`（全量读取，不按加载策略裁剪）
3. 对本集每句台词（`**角色**：台词` 格式 + △ 描述段里的引号台词）做匹配：
   - **精确命中**：本集台词与 used-lines 某条完全相同
   - **近似命中**：5 字以上连续子串匹配（如"如果有来生我还要遇见你"和"如果有来生，还会遇见你"）
4. 统计命中次数 K

### 扣分规则（写入 SKILL.md#自检 的表格硬约束）

- K = 0：0 扣分
- K = 1 或 2：-2 分（从连贯性维度扣）
- K ≥ 3：-5 分且该维度上限 3 分

### 例外放行

若本集命中的是"允许复用的例外"（口头禅同一角色 ≤3 集间隔、世界观术语、明确 callback 意图），可在 findings 中标注 `[意图复用]` 不扣分。但要求 Claude 给出具体理由，不许默认放行。

## 反模式禁区（违反则本机制失效）

1. ❌ **凭记忆冒充扫描**：Claude 回答"我记得之前有这句"而没 Read used-lines.md 实扫
2. ❌ **追加少于 3 条敷衍**：一集只追加 1-2 条（说明没认真提取锚句）
3. ❌ **把对白全量追加**：把本集所有台词都写进 used-lines.md（稀释正向召回信号，使整个机制失效）
4. ❌ **overwrite 而非追加**：用 Write 工具覆盖整个文件而非追加 section
5. ❌ **用"don't 式否定约束"框架**：Claude 对自己说"不要用 XX 台词"——改用"已用清单，从候选集剔除"框架
6. ❌ **在 used-lines.md 中标注"建议"/"优秀"等评价**：本文件只记录事实（已用），不做评价

## 首集传播句（v1.20.0 新增）

### 写入触发

`/分集 1` 完成正文后，除常规 3-5 条提取外，**额外提取 1 条** `[首集传播句]`：

- 自动 prompt 用户填一条（或 Claude 主动从已写正文中识别候选并询问确认）
- 写入 `episodes/used-lines.md` 的 `## ep001` section 末尾
- 单独标 `[首集传播句]` 类别（不与其他 7 类合并）

### 三个特征（必满）

1. **偏短**：≤ 12 字优先；硬上限 20 字
2. **角色化**：能从这句话辨认出哪个角色在说，不是 AI 通用金句腔（如 "命运不会辜负努力的人" 是金句腔，被否）
3. **可回收**：后续集合应至少 1 次打脸 / 反讽 / 镜像 / 呼应这句话

### 跨集回收检查（自检 N≥2 集时执行）

`/自检` 检查 N≥2 集时，额外扫 `used-lines.md` 中 `[首集传播句]` 条目：

- 若 N ≥ 3 集后该句仍**未被回收**（EP02 ~ EPN 之间无精确/近似命中）→ findings 中提示："考虑在本集回收首集传播句：'<原句>'"
- 软提醒，不扣分（避免误伤剧情节奏）
- 若已回收 → 不提示

### 设计意图

首集传播句要么是"打脸闭环的种子"（让观众看到反派被自己说过的话打脸），要么是"角色 ID 的 callback 锚点"（让观众形成对角色的辨识）。**光记录不回收等于死库存，不要为存而存**。

### 反例（不计为有效首集传播句）

- ❌ "我一定会成功的"（金句腔，无角色 ID）
- ❌ "这世界上没有什么是绝对的"（哲学腔，无角色 ID）
- ❌ "不可能！这绝对不可能！"（功能性惊讶台词，无独特性）
- ❌ "你这种存在感，挡路倒是一流"（够具体但 25 字，超字数上限）

### 正例

- ✅ "三寸三分，不差一毫" [陆离] —— 角色 ID 强 + 短 + 后续可专业场景回收
- ✅ "我不进会议室。" [陈九州] —— 角色 ID（不抢位）+ 极短 + 后续职位反转时回收成讽刺
- ✅ "玉佩不能碎" [陆离] —— 短 + 执念锚 + 后续玉佩成关键道具时回收

## 与其他文件的关系

- **`characters.md` 角色-语言风格映射表**：定义角色该怎么说话（风格层）
- **`setting-bible.md`**：定义世界观/专业术语（内容层）
- **`used-lines.md`**（本文件）：记录已说过的具体短语（历史层）

三者不冲突——前两者约束"应该怎么说"，本文件约束"不要说过的原话"。
