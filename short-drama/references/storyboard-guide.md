# 分镜拆解指南 | Storyboard Guide

> `/分镜` 命令执行时加载本文件。

## 景别定义

| 景别 | 英文 | 画面范围 | 适用场景 |
|------|------|---------|---------|
| 远景 | ELS (Extreme Long Shot) | 大环境全貌 | 建立场景、时间跳转 |
| 全景 | LS (Long Shot) | 人物全身+环境 | 开场交代、角色出场 |
| 中景 | MS (Medium Shot) | 腰部以上 | 对话、日常互动 |
| 近景 | MCU (Medium Close-Up) | 胸部以上 | 情绪表达、重要对话 |
| 特写 | CU (Close-Up) | 面部/关键物件 | 情绪高潮、关键道具 |
| 大特写 | ECU (Extreme Close-Up) | 局部细节（眼睛/手/嘴唇） | 悬念、心理冲击 |

### 景别分布参考（行业数据）

| 指标 | 参考值 | 来源 |
|------|--------|------|
| 近景档占比（大特写+特写+近景） | 50-65% | 好莱坞商业片标准（近景特写需达一半以上）+ 竖屏短剧美学（人物占画面 70%+） |
| 中景档占比（中景） | 25-35% | CineScale 数据集（~3500 部美国电影统计，MCU 约 28-33%） |
| 远景档占比（全景+远景） | 5-15% | CineScale（长镜头极少，约 0.4%）；竖屏短剧因 9:16 比例更少 |
| 同景别连续上限 | 2-3 个 | 中国视听语言教材 |
| 对话中空镜间隔 | 每 5-8 个镜头 | 西方剪辑惯例（StudioBinder, No Film School） |
| 建立镜头 | 每个新场景 1 个 | 通用惯例 |

### 景别跳转规则

- **相邻景别**（如特写↔近景、近景↔中景）：需以 3+ 个镜头组成「镜头组」递进过渡，不宜直切
- **跨级景别**（如特写→全景、近景→远景）：可直接剪切，视觉反差制造冲击力
- **30 度规则**：同一主体的连续镜头，机位角度至少变化 30 度或焦距变化 20mm+

## 镜头运动

| 运动 | 英文 | 效果 | 适用场景 |
|------|------|------|---------|
| 推 | Push In / Dolly In | 逐渐逼近，制造压迫感 | 揭秘、情绪升级 |
| 拉 | Pull Out / Dolly Out | 逐渐远离，揭示全貌 | 真相大白、孤独感 |
| 摇 | Pan | 水平扫视 | 环境介绍、跟随视线 |
| 俯仰 | Tilt | 垂直扫视 | 展示人物全貌、高大建筑 |
| 跟 | Tracking | 跟随主体移动 | 追逐、角色行走 |
| 环绕 | Orbit | 绕主体旋转 | 情绪爆发、对峙高潮 |
| 升降 | Crane / Jib | 垂直升降 | 开场俯瞰、结尾升空 |
| 手持 | Handheld | 轻微晃动 | 紧张、临场感 |

## 即梦 Seedance 2.0 Prompt 工程（核心）

### 工作流：全能多参为主，关键帧为辅

| 模式 | 何时用 | 输入 |
|------|--------|------|
| **全能多参**（默认） | 大部分镜头 | 角色参考图 + 场景图 + 文字 prompt，一步生成视频 |
| **关键帧（首尾帧）** | 全能多参效果不好时 | 先文生图出关键帧，再图生视频 |

全能多参用 `@` 语法绑定素材角色：
```
@图片1 作为角色参考
@图片2 作为场景参考
[镜头描述]
```

### Prompt 公式（全能多参视频）

```
{镜头运动} + {主体及动作} + {场景环境} + {光影氛围} + {风格尾缀}
```

20-50 词，一条 prompt 描述完整镜头。不拆文生图/图生视频两条。

**示例：**

镜头 1（全景，缓推，无角色）：
> 镜头从远处缓慢推近，茫茫大海上一艘铁锈斑斑的货轮破浪前行，白色浪花翻涌，远处天际线灰蓝色调，海鸥盘旋，傍晚暖光，电影质感，4K

镜头 2（近景，固定）：
> @图片1 作为角色参考。年轻女生坐在咖啡馆窗边低头看手机，手指轻滑屏幕，窗外阳光透过玻璃洒在脸上，表情略带失落，电影质感，4K

镜头 3（特写，手持）：
> 手部特写，一只手猛地攥紧文件，指节发白，纸张褶皱，桌面上散落的咖啡杯微微震动，侧光，电影质感，4K

### One-Move Rule（一动原则）

**每个镜头只描述一个核心动作。** 多动作让 AI 困惑，输出不可控。

- ✅ `她猛地转身，眼中含泪`
- ❌ `她猛地转身，抓起杯子摔在地上，然后冲出门去`

连续动作→拆成多个镜头。

### 风格尾缀（固定，所有镜头统一）

```
电影质感，4K
```

含角色时追加：
```
同一角色，服装一致，发型不变
```

**重要**：风格尾缀全剧统一，只改动作/场景部分。不要每个镜头换风格词。`电影质感` 只写一次，不重复。

### Prompt 生成规则

1. **中文优先**：即梦对中文理解更好
2. **20-50 词**：关键特征写到位即可，不堆砌修饰词
3. **物理化描述**：描述可拍摄的画面，不写抽象感受（「心痛」→「指甲掐进掌心」）
4. **避免否定句**：写"要什么"不写"不要什么"
5. **物理视角一致性**：描述内容必须在该景别/角度下可见
6. **适度留白**：过度堆砌反而让 AI 误解，不需要描述画面中每一个物件
7. **必须有主语**：不写「生长，茂盛」，写「藤蔓沿墙壁缓慢生长」
8. **不重复词**：同一 prompt 中一个词只出现一次

### 多角色场景处理

**方法 A：全能多参内多角色绑定**
```
@图片1 为女主角
@图片2 为男配角
女生转头看向男生，男生低头回避目光，
两人之间保持一臂距离，咖啡馆暖光，电影质感，4K
```
每个角色上传独立参考图，动作描述具体到每个人。

**方法 B：拆分单人镜头交叉剪辑**（更稳定）
多人同框一致性不够时，拆成单人近景/特写分段生成，后期剪辑交叉组合。

### 何时回退到关键帧模式

| 场景 | 用关键帧 |
|------|---------|
| 全能多参效果不好（人物变形、动作不对） | 先文生图定格画面，再图生视频 |
| 需要精确控制起止画面 | 首帧图 + 尾帧图 |
| 多段视频接力（>15 秒） | 上一段尾帧 → 下一段首帧 |

关键帧模式时拆为两条 prompt：
- **文生图**（30-60 词）：静态定格瞬间 = `{角色外观} + {姿态定格} + {场景} + {光影} + {风格尾缀}`
- **图生视频**（15-40 词）：动态变化 = `{镜头运动} + {主体动作} + {环境变化}`

### 常见错误

| 错误 | 解决 |
|------|------|
| 修饰词重复（「电影质感，4K，画面清晰，电影质感」） | 风格尾缀只写一次 |
| 缺主语（「翻涌，磅礴」） | 写「海浪翻涌，浪花拍打船身」 |
| 抽象描述（「氛围紧张」） | 物理化：「两人对视，男主握紧桌沿」 |
| 手部变形 | 指定「双手自然放在桌面」或避免手部特写 |
| 每段换脸 | 每段都用 `@图片1` + 一致性尾缀 |
| 素材过多（>8 个） | 精简素材，避免模型困惑 |
| 画面抖动 | 加「镜头稳定，画面平稳」 |

### Prompt 质量自检

- [ ] 是否只有一个核心动作？
- [ ] 20-50 词内？没有堆砌重复？
- [ ] 有明确主语？
- [ ] 情绪已转为物理动作？
- [ ] 风格尾缀与其他镜头一致？
- [ ] 该景别下描述内容物理上可见？
- [ ] 含角色时有 `@` 绑定或外观描述？

## 情绪→物理动作转换对照表

编剧常写情绪，但 AI 视频生成需要可拍摄的物理动作。以下为常用转换：

| 情绪/状态 | 物理动作（可直接用于 prompt） |
|----------|--------------------------|
| 愤怒 | 握紧拳头，青筋暴起 / 猛拍桌面，杯子震动 / 咬紧牙关，太阳穴跳动 |
| 悲伤 | 眼眶湿润，咬着下唇 / 低头不语，手指无意识搓衣角 / 背对人群，肩膀微微颤抖 |
| 恐惧 | 瞳孔骤缩，后退半步 / 手指微颤，攥紧手机 / 屏住呼吸，身体僵直 |
| 惊讶 | 嘴微张，手中物品滑落 / 猛地站起，椅子向后滑 / 眨眼两次，目光定格 |
| 紧张 | 反复舔嘴唇 / 手指无节奏敲桌面 / 频繁看手表 / 不自觉吞咽 |
| 心动 | 目光不自觉追随对方 / 微微侧头，嘴角上扬 / 手指无意识触碰嘴唇 |
| 得意/嚣张 | 单手插兜，下巴微抬 / 翘起二郎腿，手指弹桌面 / 后仰靠椅背，双臂枕在脑后 |
| 绝望 | 瘫坐在地，双手抱膝 / 仰头闭眼，泪痕横流 / 双手无力垂下，目光空洞 |
| 算计 | 手指交叉撑在下巴，眼神微眯 / 转动手中的笔，嘴角勾起 / 侧身倚墙，眼角余光扫视 |
| 犹豫 | 手伸向门把手又缩回 / 来回踱步 / 欲言又止，张口又闭 |
| 温柔 | 轻轻拨开对方额前碎发 / 双手捧住对方的脸 / 将外套披在对方肩上 |
| 疲惫 | 揉太阳穴，闭眼长叹 / 松开领带，瘫在沙发 / 单手撑额，咖啡凉了没喝 |
| 隐忍 | 嘴角咬出血印 / 指甲掐进掌心 / 笑着点头但手在桌下颤抖 |
| 威胁 | 缓步靠近，俯视对方 / 一只手搭在对方肩上，力度逐渐加大 / 低声说话，几乎贴着对方耳朵 |
| 释然 | 长舒一口气，肩膀放松下沉 / 仰望天空，嘴角微扬 / 缓缓松开攥紧的拳头 |
| 尴尬 | 挠后脑勺，错开目光 / 干咳一声，假装看手机 / 僵硬地笑，不知该把手放哪 |
| 震惊（重大发现） | 手中文件/手机滑落，呆立原地 / 连退几步撞到墙 / 缓缓转头，嘴唇颤抖 |
| 决心 | 深吸一口气，直视前方 / 将手中物品重重放下 / 起身整理衣领，目光坚定 |
| 嫉妒 | 握紧杯子，指节发白 / 移开视线，嘴角抿成一条线 / 将手中的东西用力撕碎 |
| 不屑 | 轻哼一声，转身就走 / 挑起一边眉毛，目光向下扫 / 弹掉衣服上不存在的灰 |

## 角色一致性三重锁定

1. **角色卡**：建立固定外观描述（年龄、发型、五官、服装），生成种子参考图
2. **`@` 绑定**：每个镜头都用 `@图片1` 引用同一张角色参考图
3. **一致性尾缀**：每段 prompt 末尾固定写 `同一角色，服装一致，发型不变`

**其他要点：**
- 避免模糊描述：不写「一个女人」，写「二十五岁中国女性，齐肩黑色直发，圆脸杏眼」
- 服装方案跟场景：换装在分镜表标注，不同场次用不同服装方案但同一场次内保持一致
- 风格尾缀全剧统一：所有镜头使用完全相同的风格关键词，只改动作/场景描述
