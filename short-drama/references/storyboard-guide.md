# 分镜拆解指南 | Storyboard Guide

> `/分镜` 命令执行时加载本文件。

## 景别定义

| 景别 | 英文 | 画面范围 | 适用场景 |
|------|------|---------|---------|
| 远景 | ELS (Extreme Long Shot) | 大环境全貌 | 建立场景、时间跳转 |
| 全景 | LS (Long Shot) | 人物全身+环境 | 开场交代、角色出场 |
| 中景 | MS (Medium Shot) | 腰部以上 | 对话、日常互动 |
| 近景 | MCU (Medium Close-Up) | 胸部以上 | 情绪表达、重要对话 |
| 特写 | CU (Close-Up) | 面部/关键物件 | 情绪高潮、关键道具 |
| 大特写 | ECU (Extreme Close-Up) | 局部细节（眼睛/手/嘴唇） | 悬念、心理冲击 |

## 镜头运动

| 运动 | 英文 | 效果 | 适用场景 |
|------|------|------|---------|
| 推 | Push In / Dolly In | 逐渐逼近，制造压迫感 | 揭秘、情绪升级 |
| 拉 | Pull Out / Dolly Out | 逐渐远离，揭示全貌 | 真相大白、孤独感 |
| 摇 | Pan | 水平扫视 | 环境介绍、跟随视线 |
| 俯仰 | Tilt | 垂直扫视 | 展示人物全貌、高大建筑 |
| 跟 | Tracking | 跟随主体移动 | 追逐、角色行走 |
| 环绕 | Orbit | 绕主体旋转 | 情绪爆发、对峙高潮 |
| 升降 | Crane / Jib | 垂直升降 | 开场俯瞰、结尾升空 |
| 手持 | Handheld | 轻微晃动 | 紧张、临场感 |

## 即梦 AI Prompt 工程（核心）

### Director's Formula（导演公式）

每个 prompt 按以下结构组织，30-80 词中文最佳：

```
{角色外观前缀} + {场景环境} + {角色动作} + {镜头运动} + {光影/氛围}
```

**示例：**
> 二十五岁中国女性，齐肩黑色直发，白衬衫黑西裤，站在落地窗前的现代办公室，双手交叉抱胸背对镜头望向窗外城市夜景，镜头从身后缓慢推近，暖黄台灯光从侧面打在她的轮廓上，氛围沉思孤独

### One-Move Rule（一动原则）

**每个镜头只描述一个核心动作。** 多动作会让 AI 困惑，输出不可控。

- ✅ `她猛地转身，眼中含泪`（一个核心动作：转身）
- ❌ `她猛地转身，抓起杯子摔在地上，然后冲出门去`（三个动作）

需要连续动作？拆成多个镜头。

### Prompt 生成规则

1. **中文优先**：即梦 AI 对中文 prompt 理解更好，不需要翻译成英文
2. **30-80 词**：太短缺细节，太长 AI 抓不住重点
3. **物理化描述**：描述可拍摄的画面，不写抽象感受
4. **自动拼接角色前缀**：如有角色卡，prompt 开头自动拼接角色的固定外观描述
5. **避免否定句**：写"要什么"而不是"不要什么"

### Prompt 质量检查

生成 prompt 后自检：
- [ ] 能否在脑中形成一个清晰的静态画面？
- [ ] 是否只有一个核心动作？
- [ ] 是否包含了镜头信息（景别或运动）？
- [ ] 是否在 30-80 词之间？
- [ ] 是否有角色外观描述（手动或角色卡前缀）？

| 愤怒 | 握紧拳头，青筋暴起 / 猛拍桌面，杯子震动 / 咬紧牙关，太阳穴跳动 |

| 恐惧 | 瞳孔骤缩，后退半步 / 手指微颤，攥紧手机 / 屏住呼吸，身体僵直 |
| 惊讶 | 嘴微张，手中物品滑落 / 猛地站起，椅子向后滑 / 眨眼两次，目光定格 |

| 得意/嚣张 | 单手插兜，下巴微抬 / 翘起二郎腿，手指弹桌面 / 后仰靠椅背，双臂枕在脑后 |

| 犹豫 | 手伸向门把手又缩回 / 来回踱步 / 欲言又止，张口又闭 |
| 温柔 | 轻轻拨开对方额前碎发 / 双手捧住对方的脸 / 将外套披在对方肩上 |
| 疲惫 | 揉太阳穴，闭眼长叹 / 松开领带，瘫在沙发 / 单手撑额，咖啡凉了没喝 |

## 角色一致性保持技巧

1. **角色卡是锚点**：每次生成包含特定角色的镜头，prompt 开头必须包含该角色的固定外观描述
2. **服装方案跟场景**：同一集内角色换装要在分镜表标注，不同场次用不同服装方案
3. **避免模糊描述**：不要"一个女人"，要"二十五岁中国女性，齐肩黑色直发，圆脸杏眼"
4. **关键帧锁定**：重要角色的首次出场镜头，先用 img2vid 工作流（生成关键帧图→图生视频），锁定角色形象
5. **批量生成时复制 prompt 前缀**：同一角色在不同镜头中，外观描述部分应完全一致（用角色卡自动保证）
