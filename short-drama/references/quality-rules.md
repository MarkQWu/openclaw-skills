# 质量规则（Quality Rules）

## 分集质量要求

- 每集 3-5 个场次
- 每集 800 字以上（中文）/ 600 words+（English）
- 景别提示：全景、中景、近景、特写（至少使用3种）
- 台词带语气或动作指示
- 每集结尾必须有悬念钩子（参考 hook-design.md）
- 第1集必须在前30秒（约前3段）抓住观众（参考 opening-rules.md）
- 付费卡点集（[付费]）结尾必须制造强悬念
- 角色称呼必须与 characters.md 称呼关系表一致（参考 realism-checklist.md）
- 角色认知转变必须有具体触发物（看到/听到/撞见），不能凭"感觉"空降结论
- 私人关系通过具体生活场景呈现，不用标签式描述（"多年好友"→具体互动）
- 情绪/立场转折必须有至少一个中间过渡状态，不能一步到位
- 正式场景（面试/谈判/汇报）台词须符合场景基本逻辑，允许戏剧化但不能违反常理

### 反抽象规则

场景描写（△ 段落）禁止使用纯情绪/氛围词。「她很难过」→「眼眶湿润，咬着下唇」；「气氛紧张」→「两人对视，男主握紧桌沿」。所有情绪必须转化为可拍摄的物理动作（参考 storyboard-guide.md 情绪→物理动作对照表）

### 台词风格校验

每句台词必须对照 characters.md 角色-语言风格映射表。霸总不会长篇大论解释心情，小白兔不会突然气场两米八

### 台词长度硬约束

单条台词最多 2 句话。超过 2 句必须拆成动作间隔（△ 插入肢体/表情描写）或对话轮次。旁白/独白例外但也不超过 3 句

### 对白与叙事语言规则（分化版）

**触发条件**：执行前读取 `.drama-state.json#mode`，`domestic`（默认/缺失）走国内规则，`overseas` 走出海规则。

#### 国内模式（mode: domestic 或缺失）

**对白格式（强制）**：

- 唯一合法格式：`**角色名**：台词内容`；对白前附语气/动作时：`**角色名**（语气描述）：台词内容`
- **禁止**双引号包裹对白内容：`"..."`（中文直角）、`"..."`（中文弯引号）、`'...'`（单引号）、`"..."`（英文直引号）——均不合法
- **例外**：角色"引用他人说过的话"的嵌套语境可用中文直角引号 `「」`（如 `**女主**：他当时说「你再等三天」`）
- **漂移防御**：△ 段落中若提到对白内容，改用间接引述（`**男主** 说要等三天`）或 `「」`

**示例**：
- ✅ `**男主**（冷声）：我等你三年了`
- ❌ `**男主**（冷声）："我等你三年了"`
- ✅ `**女主**：他说「你再等三天」，我就信了`

#### 出海模式（mode: overseas）

**对白格式（强制）**：好莱坞行业标准 `**CHARACTER NAME** (tone direction): "dialogue"` —— **必须**使用英文直引号 `"..."`包裹对白内容。这是海外市场硬门槛，不得剥离。

**示例**：
- ✅ `**JACK** (coldly): "I've waited three years for you."`
- ❌ `**JACK** (coldly): I've waited three years for you.`（缺引号，不合规）

#### 破折号规则（两种模式共用，强制）

**字符**：全角中文 `——`（两个 U+2014 连写），不得用单破折号 `—` 或 ASCII `--`

**使用场景**：

- 台词内部停顿/犹豫/被打断：`**女主**：我——我不知道`
- 场景描述补充说明：`他推开门——那扇用了三十年的木门`
- 心理独白引出：`她心里翻腾着——这人绝不能信`

**密度上限**：每集 `——` 出现 2-5 次。<2 次该集格式与可拍性维度 -2 分；>5 次同样 -2 分（避免泛滥拖沓）。

**出海模式破折号**：英文用 em dash `—`（单 U+2014）或 `——` 皆可；但同一集内保持一致。

### 爽点密度硬约束

每集至少包含 3 个爽点（参考 satisfaction-matrix.md 的 5 大类型）。写完后在集末自查清单中列出本集爽点及类型，不足 3 个不算完成

### 秒级节奏锚点

每集必须包含以下三个节奏锚点，在剧本中用 `[锚点]` 标记：
- [锚点]**0-3s 冲突点**：开场前 3 段内必须出现冲突/悬念/反转（不能用铺垫或环境描写开场）
- [锚点]**30s 情绪爆破**：中段必须有一个情绪高峰（打脸/揭秘/爆发），对应约第 2-3 场次
- [锚点]**结尾钩子**：最后一段必须是悬念钩子（已有规则，此处强调必须标记 [锚点]）

#### anchor 节奏污染追问（v1.15.0 新增）

`/自检` 检查节奏锚点维度时，如发现**缺失任一锚点**（0-3s 冲突 / 30s 爆破 / 结尾钩子）：
1. 检查 `creative-plan.md` 是否有 `anchor` 字段（v1.15.0 MVP 5 题材可能有）
2. 如有 → 在 findings 中追问用户：「本集节奏锚点缺失，是否被 anchor《[anchor 名]》节奏污染？[Y/N]」
3. 用户回 **Y** → 建议执行 `/分集 N --fix anchor-rhythm` 重写本集（临时清空 anchor 重新生成）
4. 用户回 **N** → 按常规节奏锚点扣分逻辑处理

### anchor 同人化禁用词清单（v1.15.0 新增 · F4 失败模式兜底）

防止模型把 anchor 当成"必须模仿的对象"，疯狂塞 anchor 标志性词汇/原台词。`/自检` AI Slop 维度扫描以下禁用词，**本集出现 ≥1 次扣分 + 标 `[anchor 同人化]`**（如有 `[意图复用]` 注解则放行，参考 used-lines.md 例外机制）：

| anchor | 禁用词清单 |
|---|---|
| 后宫·甄嬛传 | 「臣妾做不到啊」「贱人就是矫情」「本宫」「嬛嬛」「钮祜禄」（核心人物名 + 标志性台词）|
| 何以笙箫默 | 「要么一辈子，要么一拍两散」「默笙」「以琛」（人物名 + 名场面台词）|
| 庆余年 | 「范闲」「叶轻眉」「庆帝」+ 直接复制粘贴中国古典诗词组合 |
| 平凡的世界 | 「孙少安」「孙少平」「田晓霞」「贺秀莲」（人物名直接出现）|
| 隐秘的角落 | 「朱朝阳」「严良」「普普」「张东升」「我们一起爬山吗」（人物名 + 名梗）|

**判定逻辑**：模型借调性时**不应**在剧本里直接出现 anchor 人物名 / 标志性台词。借调性的标志是"看不出借了哪部"——隐式吸收。如剧本里出现"甄嬛"或"范闲"作为本剧人物名，明显是 anchor 串味。

## 已用台词库（跨集去重机制）

**文件**：`used-lines.md`（项目根目录，和 `episode-directory.md` 同级）。详细格式/加载/扫描/反模式规则见 `used-lines-protocol.md`（生成分集前/后必读）。

### 核心约束

- **写第 N 集前**：`/分集` 命令强制 Read `used-lines.md`（存在则读，不存在视为空）。加载策略按集数分档（N≤10 全量 / 10<N≤30 最近 10 集全量+更早仅标题及前 2 条 / N>30 更激进裁剪），详见 `used-lines-protocol.md#读取规则`
- **语用框架**：这不是"don't 式否定约束"，是**"已用清单，从候选集剔除"的正向召回**。生成情绪锚句/遗言/口头禅等类别时，优先选新表达而非复用已用清单中的
- **写完第 N 集**：`/分集` 命令强制追加 3-5 条本集"高复读风险台词"到 `used-lines.md`（Read-Modify-Write，不得 overwrite）。判定标准：情绪锚句 / 遗言告别 / 口头禅 / 3 字以上标志性形容词组合 / 特殊意象。不追加功能性台词（"走吧"、"知道了"）
- **/自检 第 5 维度**：连贯性扫描新增子项"跨集台词复用扫描"——本集关键台词 grep `used-lines.md`，命中 ≥1 次扣 2 分，≥3 次扣 5 分且该维度上限 3 分
- **允许的例外**：角色口头禅（≤3 集间隔）/ 世界观专业术语 / 明确设置的 callback 意图——由 `/自检` 在 findings 中标 `[意图复用]` 放行，但必须给具体理由

### 反模式禁区

- ❌ 凭记忆冒充扫描（必须真实 Read used-lines.md）
- ❌ 每集追加 <3 条敷衍
- ❌ 把全部对白写进 used-lines.md（稀释正向召回信号）
- ❌ overwrite 整个文件
- ❌ 用"don't 用 XX 台词"措辞

## 上下文窗口管理（50+ 集长剧）

写后续集数时，上下文不够装全部已完成集数。按以下策略分批加载：

| 内容 | 加载方式 | 始终加载 |
|------|---------|---------|
| creative-plan.md | 全文 | [完成] |
| characters.md（含语言风格表） | 全文 | [完成] |
| episode-directory.md | 全文 | [完成] |
| 最近 3 集完整剧本 | ep{N-3} ~ ep{N-1} 全文 | [完成] |
| 关键剧情集（[关键]标记） | 全文 | 按需（涉及伏笔回收时） |
| 其余已完成集数 | 仅读标题+钩子+前情提要行 | 按需 |

**执行规则**：
- 当 N ≤ 10：加载全部已完成集数（不分批）
- 当 N > 10：按上表分批，优先保证最近 3 集 + 角色档案 + 目录在上下文中
- 批量写 `/分集 5-8` 时：第 5 集写完后，将其纳入"最近 3 集"窗口，滑动前进
- 发现需要回溯远期集数（如回收伏笔）：临时加载该集全文，用完释放

## 创作原则

1. **渐进式创作：** 每一步确认后才进入下一步，不一口气生成不可控的内容
2. **可随时调整：** 任何阶段可以回头修改，修改后自动更新下游内容
3. **上下文连贯：** 写后续集数时必须参考已有内容，避免前后矛盾
4. **质量可控：** 每集写完可跑自检，不满意就重写
5. **专业格式：** 输出的是文学剧本格式，导演拿到手能直接拍

## 考据可追溯性自检流程（厚型/中型题材）

`/自检` 第 8 维度的核查流程：

1. 检查 `setting-bible.md` 是否存在
   - 不存在 + 题材为厚型 → 第 8 维度 0 分，提示「请先 /考据 auto」
   - 不存在 + 题材为中型 → 第 8 维度 0 分，但允许用户跳过（标记 `bibleScope: ["user-declined"]`）
   - 不存在 + 题材为轻型 → N/A 不计入

2. 提取剧本中所有"具体专业事实"，逐条对照 bible：
   - 专业术语 / 官名 / 制度名 / 药物名 / 法条编号 / 监管机构 / 数字精度（年份/剂量/能量参数）
   - 形式化检索：见 `quality-rubric.md#编造检测启发式` 的正则模式
3. 对每条事实判定：
   - 在 bible 且 verified → ✅ 通过
   - 在 bible 但 [待人工核源] → ⚠️ 需在剧本中加 `[考据待定]` 标注
   - 不在 bible 但应在 → ❌ 编造，扣分
   - 不在 bible 但属于「虚构白名单」→ ✅ 通过

4. 时代/领域常识扫描（按 `setting-bible.md` 时代/场景常识锚点字段）：
   - 称谓 / 饮食 / 服饰 / 礼仪 / 工具
   - 命中"已知雷区"字段 → 上限 6 分；命中 ≥2 → 上限 3 分

5. 制度规则一致性扫描：前后集出现相同制度引用时，必须遵守相同规则

详细分数锚点见 `quality-rubric.md#八-考据可追溯性`。

## 自检维度细则

8 维度检查（轻型题材跳过第 8 维度），每项 1-10 分（爽点 1-15，格式 1-5）：

| 维度 | 检查内容 |
|------|---------|
| 节奏 | 开场是否够快、有无拖沓段落、紧张-舒缓交替是否合理、**[锚点] 三锚点检查**：0-3s 冲突点是否在前 3 段内、30s 情绪爆破是否存在、结尾钩子是否标记 [锚点]——缺少任意一个锚点该维度不超过 5 分 |
| 爽点 | 数量是否足够（**硬约束：≥3 个，不足 3 个该维度不超过 4 分**）、强度是否达标、类型是否多样 |
| 台词 | 有无废话、角色区分度、是否口语化自然、**称呼是否匹配层级关系、场景台词是否符合场景逻辑**（参考 realism-checklist.md）、**每句台词是否符合角色-语言风格映射表**、**台词长度检查：逐条扫描，标出超过 2 句的台词并建议拆分** |
| 格式 | 场景头完整性、景别标注、音乐提示、特殊标记、**输出语言一致性**（全文与 `.drama-state.json` 的 `language` 一致）、**模板合规**（不含模板外区块如导演手记/创作心得） |
| 连贯性 | 与前后集是否矛盾、角色行为是否一致、伏笔是否延续、**认知转变是否有具体触发物、情绪转折是否有过渡状态、关系呈现是否生活化**（参考 realism-checklist.md） |
| 反抽象 | **△ 段落是否有纯情绪词未转为物理动作**（扫描全文，逐个标出「她很伤心」「气氛凝重」类描述并给出物理化替换建议） |
| AI Slop | **台词书面化**：扫描全部台词，标出「我认为我们应该」「在某种程度上」「不可否认」等书面表达，给出口语化替换；**情绪转变过平滑**：检查角色情绪/立场转变是否缺少抗拒、犹豫、反复等中间状态（「她听完立刻原谅了他」= AI Slop）；**巧合堆砌**：统计「正好」「恰好」「刚好」「碰巧」「没想到」出现次数，全集 ≥3 次判定为巧合过多，每处标出并建议用因果逻辑替代 |
| 考据可追溯性 | **题材分化**：厚型/中型计入总分（满分 80），轻型 N/A（满分 70）。**核查流程**：见上方「考据可追溯性自检流程」。**雷区否决**：命中 1 条雷区上限 6 分，≥2 条上限 3 分。详见 `quality-rubric.md#八-考据可追溯性` |

## --fix 模式流程

1. 先完成完整自检，生成评分报告和问题清单（与普通模式相同）
2. 逐项展示每个【严重】问题的定位和修复选项（A/B）
3. 用户对每项选择：A（应用方案A）/ B（应用方案B）/ 跳过
4. 直接修改 `episodes/ep{NNN}.md` 源文件（非副本）
5. 全部修复完成后，自动重新自检并更新分数

### 考据缺失补丁（--fix 子流程）

第 8 维度扣分项的 --fix 处理：

- **专业事实未映射 bible** → 选项 A：调用 /考据 auto 增量检索补 bible 字段；选项 B：改为模糊表述；选项 C：标 `[虚构]` 加入白名单
- **命中已知雷区** → 选项 A：按 bible「已知雷区」字段重写该段；选项 B：删除该段
- **bible [待人工核源] 字段被引用** → 选项 A：加 `[考据待定]` 标注（不扣分但提示用户）；选项 B：改为已 verified 字段

## 分数持久化

自检完成后，将每集总分写入 `.drama-state.json` 的 `qualityScores` 字段（格式：`{"1": 52, "2": 61, ...}`）。`/导出` 依赖此数据做质量门控。
