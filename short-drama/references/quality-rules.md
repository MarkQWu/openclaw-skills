# 质量规则（Quality Rules）

本文件为跨介质通用规则。执行前先读 `.drama-state.json#medium` 按介质补充加载：
- `medium="ai_live"`（默认 · 仿真人 AI 剧）→ 额外加载 `ai-live-rules.md`（场次 3-5 / 单条台词 ≤2 句 / VO ≤20% 字数 / 微表情禁用）
- `medium="comic"`（漫剧）→ 额外加载 `comic-rules.md`（场景 ≤3 / 单条台词 ≤6 句 / 场景头新格式 / OS/VO 情绪标签）

## 分集质量要求

- 每集 800 字以上（中文）/ 600 words+（English）
- 景别提示：全景、中景、近景、特写（至少使用3种）
- 台词带语气或动作指示
- 每集结尾必须有悬念钩子（参考 hook-design.md）
- 第1集必须在前30秒（约前3段）抓住观众（参考 opening-rules.md）
- 付费卡点集（[付费]）结尾必须制造强悬念
- 角色称呼必须与 characters.md 称呼关系表一致（参考 realism-checklist.md）
- 角色认知转变必须有具体触发物（看到/听到/撞见），不能凭"感觉"空降结论
- 私人关系通过具体生活场景呈现，不用标签式描述（"多年好友"→具体互动）
- 情绪/立场转折必须有至少一个中间过渡状态，不能一步到位
- 正式场景（面试/谈判/汇报）台词须符合场景基本逻辑，允许戏剧化但不能违反常理

### 反抽象 & 画面可拍性规则（轻约束）

**设计意图**：让场景叙事层（摄影机能看到的层）落地可拍，同时保留 OS/VO 层承载诗意想象力（anchor 机制的红利落在这里）。**不是关键词黑名单**——用分层原则 + 3 问自检 + 翻译示范，让模型自己判断。

#### 分层原则（4 层，约束梯度越靠近摄影机越严）

| 层 | 识别标志 | 约束 |
|---|---|---|
| **场景叙事层** | `△` 开头 / 景别标注（`WIDE SHOT` / `MEDIUM SHOT` / `CLOSE-UP`）开头 / `（BGM：...）` / `（音效：...）` / `[音乐]` 提示（老格式兼容）/ `【闪回】…【闪出】` 段落 | **必须可拍**（物件/动作/环境/表情）。**禁用第一人称感受动词**（感觉/觉得/意识到/感受到）—— 命中即判越界，强制翻译成 OS/VO 或生理反应 |
| **OS 内心独白** | `**角色**（OS）:` | 单段（连续同一 OS 块）内 **≤1 核心比喻**，不跨段累加；单段 ≤3 句；**禁止** "看不见的 X / 胸腔里" 反直觉画面 |
| **VO 旁白** | `**角色**（VO）:` | 每集 VO 总字数 ≤ 本集总字数 **20%**；单段 ≤3 句；**不得替代**场景叙事层的可拍性职责（不允许"场景层抽象 → VO 补解释"分工） |
| **对白** | `**角色**：` | 适用现行对白规则，本节不覆盖 |

**闪回/幻觉段落** 仍归场景叙事层处理——不给 mate bond 幻觉 / 梦境 / 通灵场景的"像电流穿过脊椎"类描写开逃生通道。

#### 4 类反抽象模式

场景叙事层 + OS 超阈位置，扫描以下 4 类：

| 类 | 模式识别 | 典型反例 → 翻译方向 |
|---|---|---|
| **原类** | 纯情绪/氛围词（无物理表现） | "她很难过" → "眼眶湿润，咬着下唇"<br>"气氛凝重" → "两人对视，男主握紧桌沿"（参考 storyboard-guide.md 情绪→物理动作对照表） |
| **A 抽象情绪物化** | 把不可见概念当物体描述其物理行为 | "空气已经绷紧了" → "前排孩子被家长按住肩膀，没人说话"<br>"压迫感从他身体里溢出来" → "他每下一步，前排人群就后退半步" |
| **B 通感/多层比喻** | 跨感官比喻 / 摄影机无法对应视觉的比喻 | "沉闷，像心跳" → "低音厚重，每击留三秒静默"（音色+节奏 brief）<br>"像水结成冰的过程" → "他眼中那一丝惊讶褪去，变冷。嘴角绷紧一分" |
| **C 不可见内心画面** | "看不见的 X" / "胸腔里" 等人体内部不可摄描述 | "胸腔里某根看不见的线被拉紧" → 拆层：场景留生理反应（"瞳孔骤然收缩，倒吸一口气"）+ OS/VO 承载概念（"……Mate Bond。"）|

**A 类可查 storyboard-guide.md 情绪→物理动作对照表**（有情绪词直接映射动作）。
**B/C 类需创作性翻译，无查表路径**——模型自己基于场景语境重写。

#### 3 问判定（写完 △ 段落后自检 / /自检 扫描）

场景叙事层每一句过 3 问：

1. **Q1（硬扫描）**：摄影机能拍到这句说的东西吗？（物件/动作/环境/表情？）
2. **Q2（半硬扫描）**：如果是不可见的东西（情绪/感受/关系），是否已翻译为人的外部反应？
3. **Q3（软提示，用于 --fix 建议生成）**：如果是比喻，本体和喻体都能拍出来吗？（"像干裂的墙皮" OK——人脸 + 墙皮都可感；"像水结冰" NOT OK——眼神变化无法对应"水结冰"视觉）

任一失败 → 标记 → 给翻译建议。

#### 扣分机制（单集上限 -3，参考破折号规则 L94）

- **维度 6（反抽象与镜头化）**：场景叙事层 + OS 超阈的 A/B/C 类违规每处 -1，**单集合计上限 -3**
- **维度 7（AI Slop）联动**：**单集场景叙事层比喻句 ≥4 处 -2**（解决"单句过 3 问但堆叠成灾"—— 如一集里"像一粒灰尘/像墨水洇开/像蜡烛闪烁/像烛芯烧尽"连发）；对白 + VO 超阈归此维度
- **C 类与"上帝视角"不重叠扣分**：C 类 = 单句不可见画面；上帝视角 = 跨场景叙述剧中人不可知。同一句不重复扣分

#### /自检 新增扫描规则

在现有"纯情绪词扫描"之外，新增 A/B/C 3 类识别（判定标准见上表反例）。详细判定边界见 `quality-rubric.md#六-反抽象与镜头化#纯情绪词判定边界` 的 A/B/C 追加段。

### 台词风格校验

每句台词必须对照 characters.md 角色-语言风格映射表。霸总不会长篇大论解释心情，小白兔不会突然气场两米八

### 台词长度硬约束（按 medium 分化）

单条台词长度硬约束按 `.drama-state.json#medium` 分化：
- `medium="ai_live"`（默认）→ 最多 2 句，详见 `ai-live-rules.md#单条台词长度`
- `medium="comic"` → 最多 6 句（漫剧分镜可承载长台词），详见 `comic-rules.md`

跨介质通用：旁白/独白也不超过 3 句（单段约束，见上方"反抽象 & 画面可拍性规则" L28 的 VO 单段 ≤3 句）。

### 对白与叙事语言规则（分化版）

**触发条件**：执行前读取 `.drama-state.json#mode`，`domestic`（默认/缺失）走国内规则，`overseas` 走出海规则。

#### 国内模式（mode: domestic 或缺失）

**对白格式（强制）**：

- 唯一合法格式：`**角色名**：台词内容`；对白前附语气/动作时：`**角色名**（语气描述）：台词内容`
- **禁止**双引号包裹对白内容：`"..."`（中文直角）、`"..."`（中文弯引号）、`'...'`（单引号）、`"..."`（英文直引号）——均不合法
- **例外**：角色"引用他人说过的话"的嵌套语境可用中文直角引号 `「」`（如 `**女主**：他当时说「你再等三天」`）
- **漂移防御**：△ 段落中若提到对白内容，改用间接引述（`**男主** 说要等三天`）或 `「」`

**示例**：
- ✅ `**男主**（冷声）：我等你三年了`
- ❌ `**男主**（冷声）："我等你三年了"`
- ✅ `**女主**：他说「你再等三天」，我就信了`

#### 出海模式（mode: overseas）

**对白格式（强制）**：好莱坞行业标准 `**CHARACTER NAME** (tone direction): "dialogue"` —— **必须**使用英文直引号 `"..."`包裹对白内容。这是海外市场硬门槛，不得剥离。

**示例**：
- ✅ `**JACK** (coldly): "I've waited three years for you."`
- ❌ `**JACK** (coldly): I've waited three years for you.`（缺引号，不合规）

#### 破折号规则（两种模式共用，强制）

**字符**：全角中文 `——`（两个 U+2014 连写），不得用单破折号 `—` 或 ASCII `--`

**使用场景**：

- 台词内部停顿/犹豫/被打断：`**女主**：我——我不知道`
- 场景描述补充说明：`他推开门——那扇用了三十年的木门`
- 心理独白引出：`她心里翻腾着——这人绝不能信`

**密度**：每集 `——` **默认 0 次是首选**（无下限，0 次完全合规）。上限 ≤3 次，超过扣分。

- 0-3 次：不扣分
- ≥4 次：格式与可拍性维度 **-2 分**（避免泛滥拖沓，且违反默认偏向完整句原则）

**理由**：剧本上破折号基本不需要出现。老规则的下限 2 与生成层「默认偏向完整句」原则冲突（强制每集 ≥2 次）。反转为"无下限 + 低上限"后，生成层和自检层口径一致。

**出海模式破折号**：英文用 em dash `—`（单 U+2014）或 `——` 皆可；但同一集内保持一致。

**默认偏向完整句（强制原则）**：

写台词默认写完整，破折号只在真正"必要"时使用。动笔前过 4 问：

1. **删掉破折号、让台词完整说完，戏剧张力是否还在？** → 在 = 删破折号，改完整句 + 合适语气标点（**默认答案是"在"——剧本基本不需要破折号，多数都能改完整句**）
2. **破折号后是否立即接"另一角色的动作/台词"（即真打断）？** → 不是（后接叙述 / 同角色下一句 / 另一角色的另起话题）= 假打断，改完整句
3. **停顿是否已在 Δ 动作描述中交代？**（如"压低声音 / 顿了一下 / 看了一眼"）→ 已交代 = 台词用句号断开分两行，不重复用破折号
4. **本集破折号已用 ≥1 次？** → 是 = **后续场景一律默认不用破折号**，只有严格通过前 3 问审视的才保留（剧本基本为 0 原则）

**3 种常见 AI 误用**（/自检 扫描命中扣分）：

| 误用类型 | 特征 | 改法 |
|---|---|---|
| 嘴硬式未说完 | 破折号后接另一人**另起话题**（不是真打断）| 改完整句 + 感叹号/问号 |
| 定语省略偷懒 | 破折号前省略了关键词（"这个"/"那种"/"守着这个——"）| 把关键词说完整再被打断 |
| 停顿观察重复 | Δ 动作已写停顿/压低声音/看一眼 | 台词用句号断开分两行 |

**/自检 新增扫描规则**：在现有"破折号密度"硬约束（≤3 次/集，0 首选）之外，扫描上述 3 种误用场景——每命中 1 种从"格式与可拍性"维度 -1 分（可叠加数量扣分）。

**破折号相关扣分总上限（强制）**：同一集中密度硬约束扣分（≥4 次 -2）+ 3 种 AI 误用扫描扣分（每种 -1）**合计不超过 -3 分**（即超过 -3 的部分截断）。格式与可拍性维度 /5，破折号问题最多吃掉 60%，保留 2 分给场景头/景别/音乐/制作难度等其他格式检查。

### 爽点密度硬约束

每集至少包含 3 个爽点（参考 satisfaction-matrix.md 的 5 大类型）。写完后在集末自查清单中列出本集爽点及类型，不足 3 个不算完成。

**有效爽点判定（v1.20.0 新增 · 防注水）**：本条计的是"爽点位置"，但要计入达标必须是**有效爽点**——见 `satisfaction-matrix.md#有效爽点判定` 4 选 3（痛点具象化 / 主角显性反应 / 局势变化 / 反派反应）。注水模式（仅身份暗示 / 仅道具露出 / 主角全程隐忍 / 仅 OS 内心）**不计**。第 1 集额外硬要求：至少 1 个**完整爽点弧**（4 节点：痛点 → 蓄力 → 小释放 → 余震）。位置标了 5 个但有效仅 2 个 → 本条达标，但首集硬要求不达标，仍判重写

### 秒级节奏锚点

每集必须包含以下三个节奏锚点，在剧本中用 `[锚点]` 标记：
- [锚点]**0-3s 冲突点**：开场前 3 段内必须出现冲突/悬念/反转（不能用铺垫或环境描写开场）
- [锚点]**30s 情绪爆破**：中段必须有一个情绪高峰（打脸/揭秘/爆发），对应约第 2-3 场次
- [锚点]**结尾钩子**：最后一段必须是悬念钩子（已有规则，此处强调必须标记 [锚点]）

#### anchor 节奏污染追问（v1.15.0 新增）

`/自检` 检查节奏锚点维度时，如发现**缺失任一锚点**（0-3s 冲突 / 30s 爆破 / 结尾钩子）：
1. 检查 `creative-plan.md` 是否有 `anchor` 字段（v1.23.0 全 13 题材均可能有）
2. 如有 → 在 findings 中追问用户：「本集节奏锚点缺失，是否被 anchor《[anchor 名]》节奏污染？[Y/N]」
3. 用户回 **Y** → 建议执行 `/分集 N --fix anchor-rhythm` 重写本集（临时清空 anchor 重新生成）
4. 用户回 **N** → 按常规节奏锚点扣分逻辑处理

| 平凡的世界 | 「孙少安」「孙少平」「田晓霞」「贺秀莲」（人物名直接出现）|
| 隐秘的角落 | 「朱朝阳」「严良」「普普」「张东升」「我们一起爬山吗」（人物名 + 名梗）|

**判定逻辑**：模型借调性时**不应**在剧本里直接出现 anchor 人物名 / 标志性台词。借调性的标志是"看不出借了哪部"——隐式吸收。如剧本里出现"甄嬛"或"范闲"作为本剧人物名，明显是 anchor 串味。

### AI 兜底形容词禁用词清单（v1.17.1 新增 · 非 anchor 串味）

与 anchor 同人化禁用词（上表）不同，这类 AI 兜底词是模型在**反差 / 情绪切换 / 身份伪装**场景的通用 AI 库存，不绑定某部 anchor。ep001 / 付费卡点等强情绪节点尤其高发。`/自检` AI Slop 维度扫描以下 5 类，**本集出现 ≥1 次扣分 + 标 `[AI 兜底词]`**：

| 类型 | 命中词/写法 | 改法方向 |
|---|---|---|
| 瞳孔描写 | 「瞳孔微微收缩」「瞳孔骤缩」「瞳孔一紧」「瞳孔放大」「瞳孔骤然收缩」 | 换成可见动作：「他看完，锁屏」/「他看了三秒」 |
| 眼神 N 式 | 「眼神骤然冷了一瞬」「眼神锐利如刀」「眼神如刀」「眼神锋利得像刀」「眼神冰冷」 | 换成具体动作：「他合上窗帘」/「他没看他们」 |
| 猎人 N 式 | 「猎人般的眼神」「猎人的眼神」「如狼如虎」「冰冷的、属于猎人的眼神」 | 删除比喻，用沉默 + 细节承载身份反差 |
| 囚徒比喻 | 「像个接受审判的囚徒」「像被判刑的人」「像待宰的羔羊」 | 删除"像 X"比喻句式，用身体语言：「他站着。没抬头」 |
| 嘴角 N 式 | 「嘴角没有任何弧度」「嘴角勾起一抹 X」「嘴角扬起一抹冷笑」 | 不写嘴角微表情（ai_live / comic 通用），改为沉默或单句对白 |

**判定逻辑**：这 5 类是 AI 在"隐藏身份 / 反差爽 / 情绪爆破"场景的通用兜底库存。共同问题 = 用华丽形容词替代具体画面，摄影机拍不到"瞳孔收缩"（特写只能拍眼球，拍不到"收缩"这个概念）。与 §反抽象 & 画面可拍性规则 C 类"不可见内心画面"（L45）同源，但这 5 类高度模板化，机械扫描效率 >> 3 问判定。

**扣分归属（强制，覆盖 SKILL.md 维度 6 的"同句命中 A/B/C 优先归维度 6"默认）**：本清单 5 类命中 → 归**维度 7 AI Slop**（不走维度 6 A/B/C 的 3 问判定路径），每类命中 -1，**单集上限 -3**（与维度 7 AI Slop 对齐）。同句命中本清单 + 维度 6 A/B/C 时以本清单优先，不重复扣分。

**证据来源**：2026-04-23 战神归来 ep001 v4 版在 5 场戏中集中出现 5 处（「瞳孔微微收缩」/「眼神骤然冷了一瞬」/「冰冷的、属于猎人的眼神」/「眼神锐利如刀」/「像个接受审判的囚徒」），盲评 AI 痕迹密度 10/10 卡死质感。

**向后兼容提醒**：v1.17.1 起新增本清单扫描，历史已生成集数重跑 `/自检` 可能下降 -1 至 -3 分（属预期行为，非 bug）。

### Type 6. 身份关系陈词（v1.18.1 新增 · F5 配套）

### 核心约束

- **写第 N 集前**：`/分集` 命令强制 Read `used-lines.md`（存在则读，不存在视为空）。加载策略按集数分档（N≤10 全量 / 10<N≤30 最近 10 集全量+更早仅标题及前 2 条 / N>30 更激进裁剪），详见 `used-lines-protocol.md#读取规则`

- **写完第 N 集**：`/分集` 命令强制追加 3-5 条本集"高复读风险台词"到 `used-lines.md`（Read-Modify-Write，不得 overwrite）。判定标准：情绪锚句 / 遗言告别 / 口头禅 / 3 字以上标志性形容词组合 / 特殊意象。不追加功能性台词（"走吧"、"知道了"）
- **/自检 第 5 维度**：连贯性扫描新增子项"跨集台词复用扫描"——本集关键台词 grep `used-lines.md`，命中 ≥1 次扣 2 分，≥3 次扣 5 分且该维度上限 3 分

- ❌ 每集追加 <3 条敷衍

## 创作原则

1. **渐进式创作：** 每一步确认后才进入下一步，不一口气生成不可控的内容
2. **可随时调整：** 任何阶段可以回头修改，修改后自动更新下游内容
3. **上下文连贯：** 写后续集数时必须参考已有内容，避免前后矛盾
4. **质量可控：** 每集写完可跑自检，不满意就重写
5. **专业格式：** 输出的是文学剧本格式，导演拿到手能直接拍

### 男主不互骂硬规则
<!-- 来源：Gemini方法论融合 2026-04-23 -->

**核心：** 主角绝不与反派进行对等的言语互骂。这会把主角降格为同层次的人。

| 场景 | ❌ 错误写法 | ✅ 正确写法 |
|------|-----------|-----------|
| 反派辱骂主角 | 主角反骂回去，言辞对等 | 主角沉默/轻笑，一句话切断对话 |
| 反派挑衅 | 主角被激怒，口头对战 | 主角用行动碾压，不在言语上纠缠 |
| 反派发表长篇嘲讽 | 主角逐条反驳 | 主角一个动作让反派哑口（铁证/身份揭露）|

**设计逻辑：** 言语对战 = 同一层级。主角的降维打击不在嘴上，在认知差距上。观众爽点是看主角碾压，不是看主角"吵赢了"。

## 考据可追溯性自检流程（厚型/中型题材）

`/自检` 第 8 维度的核查流程：

1. 检查 `setting-bible.md` 是否存在
   - 不存在 + 题材为厚型 → 第 8 维度 0 分，提示「请先 /考据 auto」
   - 不存在 + 题材为中型 → 第 8 维度 0 分，但允许用户跳过（标记 `bibleScope: ["user-declined"]`）
   - 不存在 + 题材为轻型 → N/A 不计入

2. 提取剧本中所有"具体专业事实"，逐条对照 bible：
   - 专业术语 / 官名 / 制度名 / 药物名 / 法条编号 / 监管机构 / 数字精度（年份/剂量/能量参数）
   - 形式化检索：见 `quality-rubric.md#编造检测启发式` 的正则模式
3. 对每条事实判定：
   - 在 bible 且 verified → ✅ 通过
   - 在 bible 但 [待人工核源] → ⚠️ 需在剧本中加 `[考据待定]` 标注
   - 不在 bible 但应在 → ❌ 编造，扣分
   - 不在 bible 但属于「虚构白名单」→ ✅ 通过

4. 时代/领域常识扫描（按 `setting-bible.md` 时代/场景常识锚点字段）：
   - 称谓 / 饮食 / 服饰 / 礼仪 / 工具
   - 命中"已知雷区"字段 → 上限 6 分；命中 ≥2 → 上限 3 分

5. 制度规则一致性扫描：前后集出现相同制度引用时，必须遵守相同规则

详细分数锚点见 `quality-rubric.md#八-考据可追溯性`。

## 自检维度细则

8 维度检查（轻型题材跳过第 8 维度），每项 1-10 分（爽点 1-15，格式 1-5）：

| 维度 | 检查内容 |
|------|---------|
| 节奏 | 开场是否够快、有无拖沓段落、紧张-舒缓交替是否合理、**[锚点] 三锚点检查**：0-3s 冲突点是否在前 3 段内、30s 情绪爆破是否存在、结尾钩子是否标记 [锚点]——缺少任意一个锚点该维度不超过 5 分 |
| 爽点 | 数量是否足够（**硬约束：≥3 个，不足 3 个该维度不超过 4 分**）、强度是否达标、类型是否多样 |
| 台词 | 有无废话、角色区分度、是否口语化自然、**称呼是否匹配层级关系、场景台词是否符合场景逻辑**（参考 realism-checklist.md）、**每句台词是否符合角色-语言风格映射表**、**台词长度检查：逐条扫描，标出超过 2 句的台词并建议拆分** |
| 格式 | 场景头完整性、景别标注、音乐提示、特殊标记、**输出语言一致性**（全文与 `.drama-state.json` 的 `language` 一致）、**模板合规**（不含模板外区块如导演手记/创作心得）；**新格式建议扫描**（见下方「新格式建议扫描」说明，0 扣分仅作【建议】标签 findings）|
| 连贯性 | 与前后集是否矛盾、角色行为是否一致、伏笔是否延续、**认知转变是否有具体触发物、情绪转折是否有过渡状态、关系呈现是否生活化**（参考 realism-checklist.md） |
| 反抽象 | **△ 段落是否有纯情绪词未转为物理动作**（扫描全文，逐个标出「她很伤心」「气氛凝重」类描述并给出物理化替换建议） |
| AI Slop | **台词书面化**：扫描全部台词，标出「我认为我们应该」「在某种程度上」「不可否认」等书面表达，给出口语化替换；**情绪转变过平滑**：检查角色情绪/立场转变是否缺少抗拒、犹豫、反复等中间状态（「她听完立刻原谅了他」= AI Slop）；**巧合堆砌**：统计「正好」「恰好」「刚好」「碰巧」「没想到」出现次数，全集 ≥3 次判定为巧合过多，每处标出并建议用因果逻辑替代。**【硬约束 v1.18.2】**：≥3 次 → AI Slop 维度 ≤6 分；≥5 次 → ≤4 分（封顶上限，与分数锚点取较低分） |
| 考据可追溯性 | **题材分化**：厚型/中型计入总分（满分 80），轻型 N/A（满分 70）。**核查流程**：见上方「考据可追溯性自检流程」。**雷区否决**：命中 1 条雷区上限 6 分，≥2 条上限 3 分。详见 `quality-rubric.md#八-考据可追溯性` |

## --fix 模式流程

1. 先完成完整自检，生成评分报告和问题清单（与普通模式相同）
2. 逐项展示每个【严重】问题的定位和修复选项（A/B）
3. 用户对每项选择：A（应用方案A）/ B（应用方案B）/ 跳过
4. 直接修改 `episodes/ep{NNN}.md` 源文件（非副本）
5. 全部修复完成后，自动重新自检并更新分数

### 考据缺失补丁（--fix 子流程）

第 8 维度扣分项的 --fix 处理：

- **专业事实未映射 bible** → 选项 A：调用 /考据 auto 增量检索补 bible 字段；选项 B：改为模糊表述；选项 C：标 `[虚构]` 加入白名单
- **命中已知雷区** → 选项 A：按 bible「已知雷区」字段重写该段；选项 B：删除该段
- **bible [待人工核源] 字段被引用** → 选项 A：加 `[考据待定]` 标注（不扣分但提示用户）；选项 B：改为已 verified 字段
